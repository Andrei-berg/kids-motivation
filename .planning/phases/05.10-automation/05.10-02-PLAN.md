From app/api/cron/missed-tasks/route.ts (the daily-cron template to mirror):
- `assertCronAuth(request)` → NextResponse|null; `const denied = assertCronAuth(request); if (denied) return denied`.
- `createAdminClient()` (service-role) — cron has no session; a cookie client returns 0 rows under RLS.
- `localDateString(now)` → family-local (UTC+3) YYYY-MM-DD "today".
- ISO day-of-week: `const jsDay = now.getDay(); const todayDow = jsDay === 0 ? 7 : jsDay`.
- schedule_items has columns: `id, child_member_id, title, type ('lesson'|'section'|'routine'), start_time (HH:MM:SS), day_of_week (int[]), is_active`.
- push resolution: `admin.from('push_subscriptions').select('member_id, subscription').in('member_id', ids)` then `sendPushToSubscription(JSON.stringify(sub.subscription), title, body, url)`.

ID mapping (CRITICAL — two different id spaces):
- `schedule_items.child_member_id` and `push_subscriptions.member_id` are `family_members.id`.
- `days.child_id`, `streaks.child_id`, `children.id` are `children.id` (a per-family TEXT id).
- Resolve the link via `family_members` rows (`role='child'`) carrying both `id` and `child_id`; build both maps before querying `days`/`streaks` by children.id and pushing by family_members.id.

From lib/services/streaks.service.ts (reuse, do NOT duplicate):
- `getSportActiveDays(childId, startDate, endDate, client)` → Set<string> of sport-active dates (pass the admin client).
- `getDayType(dateStr, isSick, vacationPeriods, childId, undefined, familyCalendar)` → { type: 'weekend'|'vacation'|'sick'|... }.
- Transparency rules from the calculators: room/sport/strong_week are transparent when `sick`; study is transparent when `weekend`|`vacation`|`sick`.
- `sendPushToSubscription(subscriptionJson, title, body, url?, icon?)`.
