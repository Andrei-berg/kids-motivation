### Gaps Summary

Two Critical, code-confirmed regressions block phase-goal achievement, both directly touching the money and data-integrity guarantees this phase promised (D-07 flag-off parity and D-09 idempotency):

1. **CR-01 (regression, all current families):** `KidDayFillForm.tsx`'s flag-off branch dropped the Room checklist section that existed before this phase. Because `day_blocks_enabled` defaults to `false`, this is live in production for every family today — kids can no longer see/toggle room tasks, and the unconditional submit path silently overwrites saved room state, permanently zeroing the room coin award. This is not a corner case; it is the default, current-day experience for the entire existing user base.
2. **CR-02 (money-minting vulnerability):** The new `custom_block` award path keys idempotency to a row UUID that a child-role session can freely delete and recreate via REST (entries RLS is intentionally family-wide, per D-07 backstop, with no role clause and no defense against this specific rotation). This defeats the `(child_id, source_type, source_id)` idempotency guarantee that SC3 explicitly requires, and constitutes an unbounded coin-minting exploit reachable by any child account.

Both were independently confirmed by direct source reading against the exact file/line citations in `05.6-REVIEW.md` (CR-01: `KidDayFillForm.tsx` line 1074 branch vs. line 943 `renderBuiltinBlock`; CR-02: `award/route.ts:569`). The `verify-day-blocks-award.mjs` script passes because it tests the DB unique-index mechanism in isolation, not the actual exploit path (source_id rotation) — a passing verify script here is not equivalent to the idempotency truth holding.

Everything else examined for this phase — the data model (SC1), the shared assembler and its vacation/schedule-day filtering (part of SC2), the flag-on award computation and byte-parity for the flag-off legacy path (part of SC3), the feature-flag mechanism and its operator-SQL-driven staged-rollout capability (SC4, per explicit context decision D-07) — is implemented, wired, and functioning as designed.
