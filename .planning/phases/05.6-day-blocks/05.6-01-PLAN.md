<objective>
Create the day_blocks data model: the config table that describes every block of a
child's day (built-in specializations + first-class custom blocks, including the 3
seeded "previously-free" activities from D-03), a completion table for custom
blocks, a per-family feature flag, and the repository that reads config / writes
completions / exposes parent CRUD. This is the foundation every other 5.6 plan
builds on — it defines the contracts (types, table shapes, source keys) that the
award route and both day-fill forms consume.

Purpose: de-hardcode the day (D-01) into data; enable per-child pricing (D-04),
multipliers (D-05), staged rollout (D-07), and custom blocks (D-02) — including
turning the stripped false-coin-promise activities into real paid blocks (D-03) —
without touching flag-off behavior.
Output: migration file applied to prod, types, repo, seed wiring.
</objective>
