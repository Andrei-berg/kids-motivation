<objective>
Create the day_blocks data model: the config table that describes every block of a
child's day (built-in specializations + first-class custom blocks), a completion
table for custom blocks, a per-family feature flag, and the repository that reads
config / writes completions / exposes parent CRUD. This is the foundation every
other 5.6 plan builds on — it defines the contracts (types, table shapes, source
keys) that the award route and both day-fill forms consume.

Purpose: de-hardcode the day (D-01) into data; enable per-child pricing (D-04),
multipliers (D-05), staged rollout (D-07), and custom blocks (D-02) without
touching flag-off behavior.
Output: migration file applied to prod, types, repo, seed wiring.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| kid client → day_blocks/day_block_entries writes | a child could try to insert/edit block config (prices) or forge completions to inflate awards |
| operator SQL → families.day_blocks_enabled | flag flip is operator-only in v1 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-056-01 | Elevation | day_blocks INSERT/UPDATE/DELETE | mitigate | RLS WITH CHECK requires role IN ('parent','extended'); kid role denied at DB. Client hides controls (UX only); RLS is the authority. |
| T-056-02 | Tampering | day_block_entries writes | mitigate | Completions writable by parent/extended only via RLS; award (Plan 03) recomputes coins server-side from the entry + config, never trusts a client amount. |
| T-056-03 | Tampering | families.day_blocks_enabled | accept | Flag flip is operator SQL in v1 (no UI write path); default false. Revisit when a self-serve toggle ships (5.8). |
| T-056-04 | Denial of Service | legacy-delete guard on FK cascade | mitigate | pg_trigger_depth() <= 1 lets COPPA family/child cascade deletes through while blocking direct legacy-block deletes (byte-copy of the room_tasks guard). |
| T-056-SC | Tampering | `npm i pg --no-save` in Task 3 | accept | pg is an already-present devDependency-class package used by existing verify scripts; no new untrusted package introduced. |
</threat_model>
