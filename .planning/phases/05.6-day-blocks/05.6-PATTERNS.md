### Block visibility / assembly (day-type × schedule-link filtering)

**Analog 1:** `lib/day-type.ts` — `getDayType()` (full file, 85 lines) is the stable input; do not re-derive day type inside the block assembler, call `getDayType` and branch on its `.type` output (`school|weekend|vacation|sick`).

**Analog 2:** `getActivitiesForDay` in `lib/repositories/expenses.repo.ts` (lines 584-609) — closest existing precedent for "filter a config list by day type + day-of-week", which is exactly D-08's block-visibility rule:
```typescript
export async function getActivitiesForDay(childId: string, date: string, currentDayType: string): Promise<ExtraActivity[]> {
  const { data, error } = await supabase
    .from('extra_activities')
    .select('*')
    .eq('child_id', childId)
    .eq('is_active', true)
    .order('sort_order')
    .order('created_at')
  if (error) throw error
  const activities = (data || []) as ExtraActivity[]

  // Convert JS day (0=Sun…6=Sat) → our system (0=Mon…6=Sun)
  const jsDay = new Date(date + 'T12:00:00').getDay()
  const dayIndex = jsDay === 0 ? 6 : jsDay - 1

  return activities.filter(a => {
    const dayTypes = a.day_types || []
    if (dayTypes.length > 0 && !dayTypes.includes('always') && !dayTypes.includes(currentDayType)) return false
    const dow = a.days_of_week || []
    if (dow.length > 0 && !dow.includes(dayIndex)) return false
    return true
  })
}
```
The `day_types` array + `'always'` sentinel and the empty-array-means-all-days convention are the exact shape to reuse for `day_blocks.day_types` (D-08). For schedule-linked blocks (section training, extra lesson), combine with the schedule-day filter from `getSectionsForDate` (`lib/repositories/expenses.repo.ts`, lines 529-543):
```typescript
const d = new Date(date)
const dayOfWeek = d.getDay() // 0=Sun,1=Mon,...,6=Sat
const DAY_KEYS = ['sun', 'mon', 'tue', 'wed', 'thu', 'fri', 'sat']
const todayKey = DAY_KEYS[dayOfWeek]

const sections = allSections.filter(s => {
  if (!s.schedule_days || s.schedule_days.length === 0) return true
  return s.schedule_days.includes(todayKey)
})
```
A schedule-linked block (D-08 "section training, extra lesson appear only on their scheduled days") should join to the linked `sections`/`schedule_items` row and reuse this same-day-membership check rather than re-implementing day-of-week math a third way.
