# Phase 04.2 Plan 04: Animation Personality (Count-up + Stagger) Summary

**One-liner:** rAF count-up for parent Dashboard coin balances + Framer Motion staggerChildren for activity feed, kid badge grid, and wallet transaction list.

## What Was Built

### Task 1: Parent Dashboard — count-up + activity stagger

**`components/parent-center/screens/Dashboard.tsx`**

- Added `useCountUp(target, duration=800)` hook using `requestAnimationFrame` with cubic ease-out (`1 - (1-t)^3`). Counts from 0 to target on every mount. Used in `ChildCard` component.
- Imported `useState`, `useEffect`, `useRef` from React (were missing — the file had no React imports).
- Imported `motion` from `framer-motion`.
- Added `listVariants` and `itemVariants` constants (outside components, module-level).
- Wrapped `activity.slice(0, 8).map(...)` in `<motion.div variants={listVariants} initial="hidden" animate="show">` with each row wrapped in `<motion.div variants={itemVariants}>`.

### Task 2: Kid achievements + wallet stagger

**`app/kid/achievements/page.tsx`**

- Imported `motion` from `framer-motion`.
- Added `listV` + `itemV` stagger variants (outside components).
- Replaced outer badges grid `div` with `motion.div` carrying grid layout styles, wrapped each badge card with `motion.div variants={itemV}`.
- Capped at `.slice(0, 12)` items.

**`app/kid/wallet/page.tsx`**

- Imported `motion` from `framer-motion`.
- Added `listV` + `itemV` stagger variants (after `QUICK_ACTIONS` const).
- Wrapped the transactions list inner container with `motion.div variants={listV}` and each `TxnRow` with `motion.div variants={itemV}`.
- Reduced from `.slice(0, 15)` to `.slice(0, 10)` to match plan spec and keep animation snappy.
- Updated `isLast` prop calculation to use `10` as cap (was `15`).

## Deviations from Plan

**1. [Rule 3 - Bug Fix] TypeScript strict-mode: ease string literal**
- **Found during:** Task 1 verification (tsc --noEmit)
- **Issue:** Framer Motion's `Easing` type doesn't accept plain `string` — `ease: 'easeOut'` fails type check
- **Fix:** Used `ease: 'easeOut' as const` in both `itemVariants` (Dashboard) and `itemV` (achievements, wallet)
- **Files modified:** All three files
- **Commit:** part of each task commit

**2. [Rule 2 - Missing React imports] Dashboard.tsx lacked useState/useEffect/useRef**
- **Found during:** Task 1 implementation
- **Issue:** The existing Dashboard.tsx had no React imports (works without them at runtime due to JSX transform, but needed for hooks)
- **Fix:** Added `import { useState, useEffect, useRef } from 'react'` alongside `motion` import
- **Files modified:** `components/parent-center/screens/Dashboard.tsx`
