## Decisions Made

- All new API functions accept familyId as an explicit parameter (not pulled from the Zustand store) — enables multi-tenant usage where caller controls scope
- Used createClient() from lib/supabase/client.ts (not the legacy lib/supabase.ts) — consistent with Phase 1.1 patterns
- seedDefaultCategories call in createFamily() is wrapped in try/catch with console.warn — seeds are convenience defaults, not required for family creation to succeed
- Legacy childId kept alongside new familyId/activeMemberId — Phase 1.4 will migrate all pages to UUIDs; premature removal would break all existing pages
