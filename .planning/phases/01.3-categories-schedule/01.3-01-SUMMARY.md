## Accomplishments

- Created `categories` table: family-scoped, supports study/home/sport/routine/custom types, sort_order, is_active, is_default flags
- Created `tasks` table: category_id FK, optional per-child assignment (child_member_id nullable), coins_reward, coins_penalty, reminder_time, notification_text, monthly_cap
- Created `schedule_items` table: unified lessons + sections + routine in one table, day_of_week INT[], start/end time, location, reminder_offset
- Created `push_subscriptions` table: JSONB subscription blob per device, UNIQUE constraint prevents duplicate endpoint storage
- Enabled RLS on all 4 tables with established family isolation pattern
- Defined `seed_default_categories(UUID)` function that inserts Учёба/Дом/Спорт/Распорядок idempotently

## Decisions Made

- Tasks 1 and 2 write to the same file sequentially; created the complete file in one atomic write since all content was well-defined
- Used `ON CONFLICT DO NOTHING` in seed function rather than `ON CONFLICT DO UPDATE` — default categories should never be silently overwritten
- The RLS pattern for `push_subscriptions` is `family_id` scoped (same as all others) rather than `member_id` scoped — allows parent to manage all family member subscriptions
