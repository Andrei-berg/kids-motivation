# Phase 1.3: categories-schedule - Context

**Gathered:** 2026-03-02
**Status:** Ready for planning

<domain>
## Phase Boundary

Parents can configure flexible activity categories with tasks and coin rules, a weekly schedule per child, and push reminder notifications. Everything is built from scratch — beautiful, polished UI (no patches to existing settings page).

</domain>

<decisions>
## Implementation Decisions

### Category structure
- Categories are shared across the whole family (one set per family)
- Tasks within categories can be assigned per-child or to all children
- Per-child tasks needed for families with kids of different ages/genders
- Default categories: Учёба, Дом, Спорт, Распорядок

### Default tasks per category
- **Учёба** — grades (5/4/3/2/1), homework checkbox, optional extra courses
- **Дом** — checklist: clean desk, made bed, clothes not scattered, similar items
- **Спорт** — coach feedback (scale: старался / молодец / не старался), section attendance
- **Распорядок** — time-based slots: wake up, breakfast, school, bedtime (21–22:00)

### Coin system
- Grade coins are configurable (not hardcoded). Starting values: 5→+50, 4→+30, 3→0, 2→-50
- Each task has a configurable coin value
- Monthly cap per task/category: child cannot earn more than N coins from that task/category in a calendar month
- Grades are "always on" — stable permanent reward not subject to monthly cap
- Распорядок: reminder/push-first for now, coins can be added later

### Streak & combo bonuses
- Streak milestones trigger automatic bonus: **7 days → +100 coins**, **14 days → +200 coins**, **30 days → +500 coins**
- Streak shield: one missed day per month is forgiven (streak doesn't reset)
- Weekly academic combo: if all grades for the week are 5s and 4s (no 3s, 2s, or 1s) → big bonus (≈50 coins / 500 rubles equivalent)
- Core philosophy: rewarding consistency builds character — system rewards duration and stability, not one-off actions

### Schedule
- Two layers: structured weekly plan (what, when, where) + push notifications per element
- Covers: school lessons (Mon–Fri), clubs/sections (with time and address), routine slots (time-based)
- One schedule per child
- Each schedule item can have a push reminder attached

### Push notifications
- Web Push API + PWA (service worker + manifest.json)
- Both parent and child receive notifications
- Child notification tone: fun and motivating — e.g. "Эй, загляни в комнату — порядок или бардак? 🏠"
- Parent notification: factual — e.g. "Адам ещё не отметил уборку комнаты"
- Notification text is customizable per task
- Reminder time is set by parent per task/schedule item
- iOS requires PWA (Add to Home Screen) — app needs manifest.json for this

### Settings / config UI
- Complete redesign from scratch — not patched onto existing settings page
- Beautiful, polished, Silicon Valley quality
- Parent-only (PIN-protected)
- Houses: category management, task CRUD, coin configuration, streak settings, schedule editor, notification settings

### Claude's Discretion
- Exact UI layout and component design for settings (redesign from scratch)
- Streak shield implementation details
- Service worker caching strategy
- Monthly cap enforcement logic (hard stop vs warning)
- Default notification texts (user gave examples, Claude can write variations)

</decisions>

<specifics>
## Specific Ideas

- "Воспитываем характер" — the system's purpose is habit formation through consistency, not just coin earning
- Coach feedback phrasing: старался / не старался / молодец / хорошо — informal, human tone
- Распорядок is a reminder system first; coins are secondary and can be added later
- Push notification tone for kids: conversational, not robotic ("Эй, загляни в комнату…" not "Reminder: clean room")
- "Как в лучших домах Силиконовой Долины" — the new settings UI should be a complete, polished redesign

</specifics>

- family_id from phase 1.1 schema scopes all category/task data
