## Decisions Made

- CoinRulesEditor and StreakSettings use `localStorage` keyed as `family_coin_rules_{familyId}` and `family_streak_rules_{familyId}` — Phase 1.3 scope only; Phase 2.1 migrates to a proper `coin_rules` table. This avoids adding a new DB column in this phase while still delivering the full UI.
- Settings page reads `familyId` from `supabase.auth.getUser()` + `family_members` query on pin success — consistent with multi-tenant pattern, no dependency on legacy Zustand store which uses string IDs (`'adam'`/`'alim'`)
- `urlBase64ToUint8Array` returns `ArrayBuffer` (via `.buffer as ArrayBuffer`) to satisfy TypeScript strict type requirements for `PushManager.subscribe({ applicationServerKey })` — Uint8Array with ArrayBufferLike is incompatible with the DOM lib's BufferSource

## Deviations from Plan

**1. [Rule 3 - Blocking] TypeScript compile requires all 7 files to exist**

- **Found during:** Task 1 verification
- **Issue:** `app/settings/page.tsx` imports CoinRulesEditor, StreakSettings, ScheduleEditor, NotificationSettings — all 4 Task 2 files. Running `npx tsc --noEmit` after Task 1 files only would fail with "Cannot find module" errors for all 4 Task 2 components.
- **Fix:** Created both Task 1 and Task 2 files before running TSC; committed them as separate commits (8c739d2 and 4ff66fb) to maintain per-task atomic commits while satisfying TypeScript.
- **Files modified:** None beyond plan

**2. [Rule 1 - Bug] urlBase64ToUint8Array Uint8Array type incompatibility**

- **Found during:** Task 2 TypeScript check
- **Issue:** `Uint8Array<ArrayBufferLike>` is not assignable to `BufferSource` for `PushManager.subscribe()` applicationServerKey. TypeScript 5.x strict mode treats ArrayBufferLike (which includes SharedArrayBuffer) differently from ArrayBuffer.
- **Fix:** Changed return type to `ArrayBuffer` with `.buffer as ArrayBuffer` cast.
- **Files modified:** components/settings/NotificationSettings.tsx
