**Daily end-of-day cron (20:00 UTC) checks routine schedule_items against days table and pushes parent notification listing missed child name and task count**

## Accomplishments
- Created GET /api/cron/missed-tasks route guarded by CRON_SECRET Bearer token
- Route queries routine schedule_items (type=routine, is_active=true) filtered by today's day_of_week using PostgREST cs operator
- Cross-checks days table to detect children with no filled day row for today
- Resolves parent member IDs via family_members table (role=parent in child's family_id)
- Fires push notifications to all parent subscriptions with child name and missed task count
- Uses Promise.allSettled so individual push failures never crash the cron job

## Decisions Made
- **Array.from() for Set/Map iteration:** TypeScript target is ES5; direct spread of Set/MapIterator triggers TS2802. Used Array.from() consistent with existing pattern in KidDayFillForm (STATE.md decision).
- **Separate parent query by family_id:** Original plan's single members query (join child + parent IDs) was simplified. New approach: query missedChildIds to get family_ids, then separately query parent members — more precise and avoids ID set ambiguity.
- **PostgREST cs filter:** `.filter('day_of_week', 'cs', JSON.stringify([todayDow]))` used instead of `.contains()` for array containment — consistent with Supabase PostgREST array operations.
