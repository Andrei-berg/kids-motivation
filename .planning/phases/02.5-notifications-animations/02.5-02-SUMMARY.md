# Phase 02.5 Plan 02: Streak Push Notifications Summary

**updateStreaks() extended to return StreakEvents; notifyStreakEvents() server action sends push to child devices on streak broken or new personal record**

## Accomplishments

- Extended `updateStreaks()` to detect and return streak events (broken/record) before writing DB update
- Created `app/actions/push-streaks.ts` with `notifyStreakEvents()` that looks up child subscriptions and sends push for each event using `Promise.allSettled`
- Wired streak event notifications into both `DailyModal.tsx` (parent-fills) and `KidDayFillForm.tsx` (child-self-fills) using fire-and-forget dynamic import pattern

## Task Commits

2. **Task 2: Server action + wire into DailyModal and KidDayFillForm** - `154b87e` (feat)

## Files Created/Modified

- `lib/services/streaks.service.ts` - Added StreakEvent/StreakEvents interfaces; updateStreak() returns event; updateStreaks() returns StreakEvents
- `lib/streaks.ts` - Re-exports new StreakEvent/StreakEvents types for backward compat
- `app/actions/push-streaks.ts` - New server action: notifyStreakEvents() fetches push_subscriptions and sends push per event
- `components/DailyModal.tsx` - Captures streakEvents, fires notifyStreakEvents fire-and-forget
- `components/kid/KidDayFillForm.tsx` - Imports updateStreaks, captures events, fires notifyStreakEvents fire-and-forget

## Decisions Made

- Dynamic import of `push-streaks` server action in client components: avoids top-level `'use server'` import boundary violations in Next.js App Router
- KidDayFillForm previously did not call `updateStreaks()` at all — added the call as part of this plan since the form needed streak event data
- `childName` parameter passed as empty string from both callers — the parameter exists in the API for extensibility but notification text does not currently include the child's name
