## Decisions Made

- ReactionPickerBar always renders inline (no long-press required) — simpler UX for web
- StickerPicker closes on outside mousedown with 50ms delay to avoid same-click dismiss
- Sticker emoji stored in `content` column — direct render without lookup table needed
- Separate Realtime channel for reactions vs messages avoids payload type confusion
- DELETE RLS policy requires explicit `id::text` cast because family_members.id is UUID in Postgres but member_id in chat_reactions is TEXT — type mismatch silently blocked all deletes
- Reactions fetched on mount (not just via Realtime) so counts survive page refresh

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Cast family_members.id to text in DELETE RLS policy**
- **Found during:** Post-Task 2 testing (reaction deletion silently failed)
- **Issue:** RLS DELETE policy compared TEXT member_id to UUID id — Postgres type mismatch caused silent failure, making reactions impossible to remove
- **Fix:** Changed subquery to `SELECT id::text FROM family_members WHERE user_id = auth.uid()`
- **Files modified:** supabase-schema-chat-reactions.sql
- **Verification:** Reaction removal confirmed working in Supabase SQL Editor and UI
- **Committed in:** `c010f4a`

**2. [Rule 1 - Bug] Load reactions on mount for persistence across refresh**
- **Found during:** Post-Task 2 testing (reaction counts were zero after page reload)
- **Issue:** ChatThread only updated reactions via Realtime INSERT/DELETE events; no initial fetch meant the reactions state map was always empty on first render after a refresh
- **Fix:** Added `getReactions(messageId)` calls after messages load on mount to populate the reactions state map
- **Files modified:** components/chat/ChatThread.tsx
- **Verification:** Reaction counts confirmed present immediately after page reload
- **Committed in:** `507fe18`

---

**Total deviations:** 2 auto-fixed (both Rule 1 — Bug)
**Impact on plan:** Both fixes essential for correct behavior (delete toggle, cross-refresh persistence). No scope creep.

## Issues Encountered

Two Supabase-specific issues discovered during testing: (1) RLS type mismatch (UUID vs TEXT) silently blocked reaction deletes — required explicit cast in policy; (2) Realtime-only state meant counts did not survive page refresh — required initial fetch on mount. Both resolved with targeted fixes before human verification.
