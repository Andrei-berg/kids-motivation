## Decisions Made
- `postSystemMessage` uses `sender_role: 'parent'` to satisfy the DB check constraint — system is not a child role
- `wallet.family_id` is available directly on the already-fetched Wallet object, so no extra DB query needed for familyId in wallet hook
- `awardBadge` already queried `children` for XP/level — extended the `.select()` to also return `family_id` and `name` to avoid a second round-trip
- Streak milestone condition: `event?.event === 'record' && familyId && [7, 14, 30].includes(current)` — only fires when a new best is set AND current count is exactly a milestone
