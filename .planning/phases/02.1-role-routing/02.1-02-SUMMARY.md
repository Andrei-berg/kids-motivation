## Decisions Made
- Used a single `supabase.from('family_members').select('id, role')` query per request. The root path `/` is classified as `isPublicPath` which would normally skip the guard block — solved by placing the root redirect INSIDE the `if (user)` block that already performs the query, so one query serves both root redirect and protected-path guard.
- Fail-open on query errors: only redirect to `/onboarding` when `!membershipError && !membership`. Transient DB errors do not lock users out.
- `isFamilyPath` intentionally not declared as a boolean variable — `/family/*` has no guard logic, so the classifier is documented via a comment but not used in any conditional.

## Deviations from Plan

None - plan executed exactly as written.

The plan specified "single DB query per request" which required careful structuring since `/` is a public path but still needs role data. Solution: place root redirect inside the authenticated-user block that performs the query rather than after it. This matches the plan's logic flow (step 6 of the action spec) while satisfying the single-query constraint.
