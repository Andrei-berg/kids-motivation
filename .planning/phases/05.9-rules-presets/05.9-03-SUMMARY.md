## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing Critical / correctness bug] `05.9-02` failed on first apply**
- **Found during:** Task 1, applying `05.9-02-subject-grades-widen.sql`
- **Issue:** `ALTER TABLE ... ALTER COLUMN grade TYPE TEXT USING grade::text` ran before `DROP CONSTRAINT IF EXISTS subject_grades_grade_check`. Postgres validates a column's existing CHECK constraints against the NEW type during `ALTER COLUMN TYPE`, even though the constraint drop was a later statement in the same batch. `grade >= 2` has no `text >= integer` operator, so it threw `42883: operator does not exist: text >= integer`.
- **Fix:** Swapped statement order — `DROP CONSTRAINT IF EXISTS subject_grades_grade_check` now runs first, then `ALTER COLUMN grade TYPE TEXT`.
- **Files modified:** `supabase/migrations/05.9-02-subject-grades-widen.sql`
- **Verification:** Confirmed via `information_schema.columns` and `pg_constraint` readback (below) that the fixed migration applied cleanly and prod was left in the pre-migration state after the first failed attempt (implicit-transaction rollback — no partial damage).
- **Committed in:** `f25a7cc`
