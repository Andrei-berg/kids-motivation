# Phase 05.9 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — custom design system (not shadcn) |
| Preset | not applicable |
| Component library | Custom, in-repo. Parent (ink) surfaces: `Card`/`Btn`/`Pill`/`Icon`/`Tabs`/`Toggle`/`Field` from `components/parent-center/ui.tsx`. Cross-theme atoms: `Amount`/`StatusChip`/`LedgerRow`/`Tabs`/`Tick`/`Stamp`/`useCountUp` from `components/design/atoms.tsx` (theme-aware via `theme: 'paper' \| 'ink'`) |
| Icon library | None external — emoji glyphs (see `EMOJIS` picker in `RoomTasksManager.tsx`) + a hand-built inline-SVG `Icon` component (`components/parent-center/ui.tsx:41`) |
| Font | Bitter (display, `T.fHead`/`base.fontDisplay`) · Golos Text (body, `T.fBody`/`base.fontBody`) · JetBrains Mono (numerals, `T.fMono`/`base.fontMono`) — loaded via `next/font`, Cyrillic subsets included |

**Rationale for "Tool: none":** this codebase deliberately built its own token/atom system across Phases 5.3, 5.7, and 5.8 (`lib/design/tokens.ts` → `paper` (kid, light) and `ink` (parent, dark) themes, re-exported through `components/kid/design/tokens.ts` and `components/parent-center/tokens.ts`). Introducing shadcn now would fork the design system mid-milestone and contradict CLAUDE.md's Styling section. All new UI in this phase must consume the existing tokens/atoms — no new hex values, no new component primitives outside what's listed above.

**Theme split for this phase:** Rule-preset picker, grade-scale editor, and Behavior Tags manager live in **Parent Center → Settings** → use the **ink** theme (`components/parent-center/tokens.ts`). The child-proposes-a-tag control lives in **`KidDayFillForm`** (kid day-fill) → use the **paper** theme (`components/design/atoms.tsx` default). Any shared atom (`Amount`, `StatusChip`, `Tabs`) must be passed the correct `theme` prop per surface — do not hardcode ink hexes into a paper-theme screen or vice versa.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-button internal gaps, dot leaders |
| sm | 8px | Row internal gaps (icon↔label↔value), stacked-card gaps |
| md | 16px | `Card` default padding (`pad={16}`), section spacing between cards |
| lg | 24px | Screen-level vertical rhythm between major settings sections |
| xl | 32px | Reserved — not expected in this phase's compact settings rows |
| 2xl | 48px | Reserved — not expected in this phase |
| 3xl | 64px | Reserved — not expected in this phase |

Exceptions (byte-match required for visual parity with sibling settings managers already shipped in this same Settings screen):
- **10px** gaps/padding — e.g. `RuleRow`/manager-row internal padding `'8px 0'`→`10px`, `RoomTasksManager` row padding `'10px 12px'`. New components (`RulePresetsCard`, `BehaviorTagsManager`, grade-scale editor rows, diff-preview rows) MUST reuse these exact values, not round to the nearest 8pt token, so they sit visually identical to `RoomTasksManager`/`CoinsRulesTab`/`PeriodsManager` already in the same tab list.
- **14px** border-radius (`T.rM`) and **12px** (`T.rL` default `Card` radius) — token-named radii, not raw spacing, but listed here since they recur at every row/card boundary in this phase.
- **26×26px** icon-buttons (move up/down/toggle/delete) in `BehaviorTagsManager` — copy `iconBtn` from `RoomTasksManager.tsx:287` verbatim; this is a declared 44px-touch-target exception because these are inline row-utility icons following the exact precedent already shipped, not primary tap targets. Primary actions (Apply preset, Approve/Reject, Save) stay ≥44px (`Btn size="lg"` = `h:44`).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label (uppercase section header) | 12px | 600 | 1.3, letter-spacing 0.08em, uppercase |
| Body (row text, tag names, descriptions, preset card copy) | 14px | 600 | 1.5 |
| Heading (card/manager title — "Rule Presets", "Grade Scale", "Behavior Tags") | 16px | 800 | 1.2 |
| Display (large mono emphasis — diff-preview changed values, grade-coin value) | 22px | 800 | 1.1, `fontVariantNumeric: tabular-nums` |

Two-weight system for this phase's new UI: **600** (regular-equivalent — labels, body, tab text) and **800** (bold — titles, primary buttons, emphasized numeric values). This matches the codebase's existing floor (nothing lighter than 500/600 is used anywhere in Settings) and its existing ceiling (800 for headings/CTAs) — do not introduce 400 or 900.

Pre-existing atom-internal exception (do not re-litigate, just reuse as-is): `Amount` sizes its own weight per `size` prop (`sm`:700, `md`/`lg`/`xl`:800) — when this phase renders coin values via `<Amount/>` (grade-coin-map rows, tag prices, diff-preview deltas), use `size="md"` for in-row values and `size="lg"` for the diff-preview's emphasized before/after numbers. Never hand-roll a coin value in a raw `<span>`.

## Color

Two theme instances apply in this phase — declare both.

### Parent Center Settings (ink theme — presets, grade scale, tag manager, approval queue)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `T.bg1` `#141021` (`ink.bg`) | Screen background |
| Secondary (30%) | `T.card` `#1D1830` / `T.cardHi` `#232040` (`ink.card`/`ink.cardHi`) | Cards: preset cards, tag rows, diff-preview panel, approval-queue rows |
| Accent (10%) | `T.indigo` `#6C5CE7` / `T.indigoHi` `#8B7BF5` (`ink.accent`) | Selected preset card border + subtle glow; active grade-scale tab (via `Tabs` accent-on-active); "Apply preset" primary `Btn`; selected/focused input border |
| Destructive | `T.danger` `#FF8A96` (`ink.danger`) / `T.dangerSoft` `rgba(255,138,150,0.14)` | Delete-tag icon button; "Reject" button in approval queue; negative-price display border/emphasis on penalty tag rows; delete-confirmation copy |

### Kid day-fill (paper theme — propose-a-tag control + pending status)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `paper.bg` `#FAF3E9` | Screen background |
| Secondary (30%) | `paper.card` `#FFFFFF` | Day-fill section card housing the behavior-tag picker |
| Accent (10%) | `paper.accent` `#5B4BD4` | Selected tag chip border/check state (mirrors `Tick` component's `on` styling) |
| Destructive | `paper.dangerText` `#B33846` | Not used for direct destructive action on kid side (children cannot delete tags) — reserved only if a negative-price tag's value is shown to the child (render via `Amount` with `money={false}` in `dangerText`, not raw hex) |

Accent reserved for: selection/active/focus state only (preset card selection, tab active state, chip selection, "Apply"/"Propose" primary buttons) — **never** for coin figures. Gold (`goldText`) stays the sole money color (existing atom law in `components/design/atoms.tsx` header comment) — every grade-coin value, tag price, and diff-preview delta renders through `<Amount money={true}/>`, which already resolves gold automatically. Do not add indigo or any new hex to a money figure.

`warning` tone (`T.warningSoft`/`ink.warning` on parent side, `paper.warning`/`warningText` on kid side) is reserved for the **pending** behavior-tag status only — reuse the exact `StatusChip tone="warning"` pattern already shipped for pending shop purchases (`app/kid/(app)/shop/page.tsx:215`, `kidShop.statusPending`). Do not invent a new "pending" color.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA — apply a rule preset (Settings) | **"Apply preset"** — shown only after a card is selected and the diff preview (below) has been reviewed; disabled/greyed until a preset is selected (mirrors `PeriodsManager`'s `disabled={!selectedPresetId \|\| applyingPreset}` pattern) |
| Primary CTA — onboarding preset step | **"Continue"** — enabled the instant a card is tapped; no separate "Apply" wording needed in onboarding (D-04: tap-card-and-done, no numbers shown) |
| Preset card labels (3 cards, same visual weight) | **"Classic"** — subhead: *"Grades and behavior both have consequences — good effort earns coins, poor effort costs them."* / **"No Penalties"** — subhead: *"Only good effort earns coins. Nothing is ever taken away."* / **"Bonuses Only"** — subhead: *"Same as No Penalties — coins are always a reward, never a deduction."* (D-01: the last two share one underlying value set; keep their subheads distinct enough to justify two cards without implying a numeric difference that doesn't exist) |
| Diff-preview heading (before applying a preset) | **"Review changes before applying"** — followed by a per-field old→new table (e.g. `Grade 3 · −3 → 0`), using `LedgerRow`-style rows with the changed value in `Amount`/`penalty` or `Amount`/`earn` tone so the direction of change is visually obvious, not just numeric |
| Diff-preview confirm / cancel | Confirm: **"Apply these changes"** · Cancel: **"Keep current rules"** |
| Grade-scale picker label | **"Grading scale"** — options rendered as a 3-way `Tabs`: **"5-point"** / **"12-point"** / **"A–F"** |
| Grade-scale switch confirmation (no data loss, but make the forward-only behavior explicit) | *"Past grades keep their original value. New grades will use the {scale} scale."* — informational, not a blocking confirm (D-08: switching is safe, no lock needed, so this is a toast/notice, not a modal) |
| Behavior Tags manager title | **"Behavior Tags"** — subtitle: *"Add tags for things worth rewarding or correcting — each has its own coin value."* (mirrors `RoomTasksManager`'s title/subtitle pattern exactly) |
| Behavior Tags — add-tag CTA | **"+ Add tag"** (mirrors RoomTasksManager's `+ {t('common.add')}` button) |
| Behavior Tags — empty state | Heading: *"No behavior tags yet"* — Body: *"Add your first tag — e.g. \"Helped a sibling\" (+5) or \"Rude to a parent\" (−10)."* |
| Behavior Tags — delete confirmation | *"Delete tag \"{tagName}\"? Past marks using this tag keep their recorded coins — only future use of this tag is removed."* |
| Behavior Tags — legacy/system tag lock | **"🔒 Default tag"** (mirrors `settings.roomTasksManager.systemTask`) — shown on the seeded "Good behavior" tag, not deletable, matching the room-tasks legacy-lock pattern |
| Kid-side — propose a tag CTA | **"Propose a tag"** (button inside `KidDayFillForm`'s behavior section) |
| Kid-side — pending status | **"Waiting for parent"** — rendered via `StatusChip tone="warning" theme="paper"`, exact reuse of `kidShop.statusPending`'s visual pattern; no coins shown yet (`LedgerRow tone="pending"` if listed in a running total) |
| Parent-side — approval queue empty state | Heading: *"No tags waiting for review"* — Body: *"When {childName} proposes a behavior tag, it'll show up here."* |
| Parent-side — approval queue actions | **"Approve"** (`Btn variant="success"`) / **"Reject"** (`Btn variant="danger"`) — one row per pending mark, showing child name, tag name+icon, and the tag's coin value via `Amount` |
| Error state (settings save / preset apply / tag CRUD failure) | *"Couldn't save — check your connection and try again."* — rendered in the existing inline error banner pattern (`T.dangerSoft` background, `T.danger` border/text, e.g. `RoomTasksManager.tsx:139`), not a toast-only failure |
| Destructive actions in this phase | 1) **Delete a behavior tag** (non-legacy only) → inline `confirm()` per existing `RoomTasksManager`/`PeriodsManager` convention, copy above. 2) **Reject a proposed behavior tag** → no separate confirm dialog (mirrors shop-purchase reject, which is also a single tap — rejection is reversible in spirit since the child can re-propose) |
