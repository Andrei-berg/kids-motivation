### Gaps Summary

Three blocker-severity gaps prevent full goal achievement:

**Gap 1 (Blocker): lib/day-type.ts returns Russian labels**
The getDayType() function returns Russian strings ('Болеет', 'Выходной', 'Учебный') that flow into DailyModal L537 and are rendered as the day-type header label. This is a lib/ utility file that was not in any plan's `files_modified` list — the plans localized the components that consume this data but not the data source itself.

**Gap 2 (Blocker): badges.service.ts BADGES array has Russian titles**
The BADGES constant (used by getAvailableBadges()) contains hardcoded Russian title and description strings. kid/achievements/page.tsx renders `badge.title` directly at L219. This lib/ file was not in any plan's scope. Badge titles on the achievements page remain Russian regardless of language setting.

**Gap 3 (Blocker): AuditLogViewer formatDate() misses two branches**
The formatDate() helper uses t() for most time ranges but L116-117 fall through to hardcoded 'Сегодня'/'Вчера' strings. The auditLogExtra namespace in both locale files already has justNow/minsAgo/hoursAgo keys — 'today' and 'yesterday' keys just need to be added and wired.

**Gap 4 (Warning): app/parent/shop/page.tsx STARTER_TEMPLATES**
16 Russian-language sample reward titles/descriptions are shown to parents when no rewards exist in the DB. This is visible UI content that doesn't switch with the language toggle.

**Gap 5 (Warning): layout.tsx metadata**
Plan 01 explicitly specified updating the metadata title to English ("FamilyCoins — Kids Motivation") but the Russian strings remain at L10-11. This affects browser tab title and SEO/PWA metadata.
