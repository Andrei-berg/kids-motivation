## Decisions Made
- Used custom React context instead of next-intl — app needs no routing-level language switching, just client-side toggle. Zero new npm dependencies.
- Browser language auto-detect: `navigator.language.startsWith('ru') ? 'ru' : 'en'` with `typeof window` guard for SSR safety. Server-side defaults to 'ru' (app's original language).
- `html lang` attribute is static `"en"` — runtime language is controlled by LanguageProvider context, not the HTML attribute. This is correct for client-side-only i18n without Next.js i18n routing.
- Missing key fallback returns the key string itself — prevents blank UI if a key is forgotten.
