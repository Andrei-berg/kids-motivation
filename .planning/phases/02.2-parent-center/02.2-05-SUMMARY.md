# Phase 02.2 Plan 05: Parent Shop Catalogue CRUD — Summary

**One-liner:** Shop catalogue CRUD page at /parent/shop — list, create, edit, delete rewards with starter templates and auto-approve toggle.

---

## What Was Built

`app/parent/shop/page.tsx` — a fully functional parent shop management page with:

- **Item list**: loads all family rewards via `getRewards({ activeOnly: false })`, shows icon, title, price badge, category chip, for_child label, and description
- **is_active toggle pill**: inline green/gray pill per card — calls `updateReward(id, { is_active: !item.is_active })` without opening the form
- **Create/Edit form**: slide-in panel with fields for icon (emoji), title, description, category select (virtual/material/experience/money), price_coins, for_child, auto_approve checkbox, is_active checkbox (edit mode only)
- **Delete flow**: trash button sets `deletingId` state → shows inline "Удалить X? [Да] [Отмена]" confirmation → calls `deleteReward(id)` on confirm
- **Starter templates**: 5 preset items (Кино, Игра на телефоне, Поздний отбой, Пицца, Поездка в парк) — loads with window.confirm then `Promise.all(addReward(...))` — button disables after success
- **Skeleton loading**: 3 `animate-pulse` placeholder cards during initial fetch
- **Empty state**: instructional message with 🛍️ icon

---

## Required DB Migration

The `auto_approve` toggle is wired in the UI and passed through `addReward`/`updateReward`, but the column does not exist in the current schema. Run this in Supabase SQL Editor:

```sql
ALTER TABLE rewards ADD COLUMN IF NOT EXISTS auto_approve BOOLEAN DEFAULT false;
```

Until this migration runs, the auto_approve value will not persist (Supabase may ignore the unknown field).
