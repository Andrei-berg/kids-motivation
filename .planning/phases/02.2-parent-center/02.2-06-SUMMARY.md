## What Was Built

### PIN Gate (REQ-SEC-002)
- `/parent/settings` shows a PIN entry screen before any settings content
- First visit (no PIN stored): two-step PIN creation flow (enter → confirm)
- Subsequent visits: PIN verification against SHA-256 hash stored in localStorage under `parent_pin_hash`
- 4-digit PIN with live dot indicators; auto-submits on 4th digit (no button needed)
- Incorrect PIN shows "Неверный PIN" in red; clears input for retry
- "Изменить PIN" link at page bottom restarts the creation flow

### Coin Rules Editor (REQ-COIN-001)
- Loads current values from `getWalletSettings()` on mount
- Editable fields: grades 5/4/3, room task, good behavior, exercise, base exchange rate
- Number inputs (min=0, max=50) with centered styling
- "Сохранить" calls `updateWalletSettings(form)` — upserts wallet_settings table row id='default'
- 2-second "Настройки сохранены" success banner in green-400

### updateWalletSettings Function
- Added to `lib/repositories/wallet.repo.ts` after `getWalletSettings`
- Upserts `wallet_settings` table with `id='default'`, spreads updates, sets `updated_at`
- Re-exported from `lib/wallet-api.ts` for use by other pages

### Audit Log Viewer (REQ-SEC-006)
- Loads children list; provides a select dropdown to switch between children
- Calls `getAuditLog(selectedChildId, 30)` — mapped from wallet_transactions
- Each row shows: icon, description, action_type chip, coins change (green/red), date (dd.MM.yyyy HH:mm)
- "Загрузить ещё" pagination appends next 30 entries
- Empty state: "Действий пока нет"
