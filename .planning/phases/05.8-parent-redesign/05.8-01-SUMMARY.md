# Phase 05.8 Plan 01: Shared logic + i18n foundation Summary

**Pure day-of-week converter, date-range days repo read, and Weekly Summary coin/completion/streak helpers — plus every 5.8 i18n key landed in en.json/ru.json up front so downstream screens only read keys, never write locale files.**

## Performance

- **Duration:** ~25 min
- **Started:** 2026-07-21T02:35:00Z (approx)
- **Completed:** 2026-07-21T02:49:45Z
- **Tasks:** 3/3 completed
- **Files modified:** 8 (2 created new modules, 3 created test files, 3 modified: day-blocks.ts, calendar.repo.ts, 2 locale files)

## Accomplishments
- `scheduleDowToBlockDow` fixes the day-of-week off-by-one at the source (schedule_items 1=Mon..7=Sun → day_blocks 0=Mon..6=Sun), tested against Mon/Sun boundaries and a mid-week day
- `getDaysInRange` gives the calendar grid a date-range days read that mirrors `getWeekData`'s exact query shape, verified via a chainable Supabase mock asserting each `.eq/.gte/.lte/.order` call
- `lib/weekly-summary.ts` locks in the D-08 correctness fix: `sumWeeklyCoins` sums only positive `coins_change`, preventing downstream screens from reintroducing the getWeekScore-based undercounting bug
- All 5.8 copy (Day Constructor schedule-link/override UI, Year Calendar aria/today/sick-overlay, Weekly Summary card) landed in both locales at identical key paths, removing the i18n write bottleneck for the parallel plans that consume these screens

## Decisions Made
- New `analytics` top-level i18n namespace created since it didn't previously exist (Analytics.tsx has no i18n usage yet) — `analytics.weeklySummary.*` is distinct from the unrelated existing `settings.<scheduleEditor>.weeklySummary` string (a schedule-editor heading, different feature)
- `dayBlocksManager.addBlock` added as a genuinely new key (plan described it as an "update" but no such key existed — only `addPlaceholder`/`namePlaceholder` did); treated as Rule-1-adjacent correction to match the plan's stated intent (CTA copy present, matching the UI-SPEC's exact "+ Add block" / "+ Добавить блок" strings)
- Aria/tab/chip copy not given verbatim by UI-SPEC's Copywriting Contract table (`byChild`, `byBlock`, `familyDefault`, `custom`, `resync`, `today`, `filterFamily`, `sickOne`, `sickMany`, and the five `aria*` day-block-row labels) authored to match the family-bank voice and the Accessibility section's aria-label examples (e.g. "Move block up"/«Переместить блок вверх»); downstream plans consuming these keys should treat the visible-copy ones (byChild/byBlock/familyDefault/custom/resync/today/filterFamily) as adjustable if a later UI-SPEC iteration specifies exact wording
