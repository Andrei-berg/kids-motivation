**Analytics screen gets a real "This week" card — coins earned (gold, from summed positive wallet_transactions), a completion-rate StatusChip, and a >=5-day streak highlight — computed once in ParentCenter's existing data-load batch and passed down as props.**

## Accomplishments
- ParentCenter.tsx now fetches a week-scoped `getTransactions(undefined, 500, weekStartISO)` batch alongside its existing loadAll() Promise.all calls (separate from the 30-row activity-feed cap), and computes `coinsThisWeek` (via `sumWeeklyCoins`, positive `coins_change` only — not the stale `getWeekScore.total`), a family-wide `taskRate`, and a `streakHighlight` (via `topStreak`, >=5-day threshold) from data already loaded per child.
- Analytics.tsx renders a new `WeeklySummaryCard` above the KPI grid: gold `Amount` (size lg) for coins, a `StatusChip` (tone via `completionTone`) for the task rate, and a `T.warning`-colored streak highlight or `T.muted` "no streak yet" fallback — with dedicated card-level empty state (first-week-of-use, both facts at 0) and error state (week-tx fetch failed) so the card never shows a blank gap.
- The existing mocked BarChart/LineChart below the new card are untouched — confirmed via isolated diffs (`git show` per task commit).
