# Phase 05.8 Plan 04: Parent-center neon-hex cleanup Summary

**Closed the last two hardcoded-neon holdouts on the parent-center dark theme — ChatPanel's DARK_ACT activity-color map and ui.tsx's cyan Btn hover — replacing all six neon hex literals with T-token references.**

## Performance

- **Completed:** 2026-07-21T02:50:00Z
- **Tasks:** 2/2 completed

## Accomplishments
- `ChatPanel.tsx`'s `DARK_ACT` map (`coins_gain`/`coins_loss`/`streak`/`badge`/`generic`) now sources `bg`/`border`/`text` entirely from `T` tokens (`T.successSoft`/`T.dangerSoft`/`T.warningSoft`/`T.indigoSoft` for fill/border, `T.success`/`T.danger`/`T.warning`/`T.indigoHi` for text) — no more `#00E676`/`#FF8A80`/`#FFD93D`/`#FFC107`/`#8B7BF5`
- Streak and badge activity types both map to `T.warning` (never gold — consistent with the "gold is money-only" rule)
- Two inline neon literals also recolored: the `activity` tab label (`#F59E0B` → `T.indigoHi`) and the badge-of-the-day label (`#FFC107` → `T.warning`)
- `glow` box-shadows kept but rebased their rgb triplet on the new token hue (T.warning's `rgb(232,147,74)`) instead of the old neon rgb
- `ui.tsx`'s `cyan` Btn variant `hBg` folded from `#4DDCFF` into `T.indigoHi`; variant name and `bg`/`fg`/`bd` left untouched (sole consumer: `ChildProfile.tsx`'s Freeze button)
