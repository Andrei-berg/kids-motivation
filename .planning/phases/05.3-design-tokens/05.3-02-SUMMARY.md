## Accomplishments

- Created `components/design/atoms.tsx` (288 lines) exporting `Amount`, `StatusChip`, `LedgerRow`, `useCountUp`, `Stamp` — all theme-aware via an explicit `theme: 'paper' | 'ink'` prop resolved by an internal `resolve()` normalizer; zero hardcoded new-system hexes (the only literal is the plan-specified `rgba(255,255,255,0.10)` ink leader).
- `Amount`: JetBrains Mono + `fontVariantNumeric: 'tabular-nums'`, letterSpacing −0.3, sm/md/lg (13/15/22px, 700/800/800), `toLocaleString('ru-RU')`, `signed` '+' only for value > 0. Gold (`goldText`) ONLY when `money` is true — the gold-for-money invariant (T-053-04) lives here and in LedgerRow's earn tone exclusively.
- `StatusChip`: soft pill (h22, r999, '0 9px', fontBody 11/600) — background is the theme tone at 14% alpha, foreground is the theme's TEXT-SAFE variant, never the raw base color.
- `LedgerRow`: the brand signature — name (ellipsized) + flex-1 dotted leader sitting on the text baseline (empty baseline-aligned flex item, −2px optical nudge) + `Amount`. Tones: earn → gold money; spend → theme text; penalty → text-safe danger; pending → text-safe muted with a "≈" prefix. Optional muted `sub` second line.
- `useCountUp(target, duration=700)`: rAF cubic ease-out (`1 - Math.pow(1 - t, 3)`, mirroring kid AnimatedNum / parent Dashboard), counts from 0 on mount then between value changes; returns `target` immediately under `prefers-reduced-motion: reduce` (T-053-05).
- `Stamp`: ~450ms translateY(8px)+scale(0.96) entrance; keyframes injected once via a guarded `<style id="fc-stamp-kf">` and wrapped in `@media (prefers-reduced-motion: no-preference)` so reduced-motion users never get keyframes at the CSS layer; a JS check additionally drops the animation property post-mount (hydration-safe). `trigger` replays via key remount.

## WCAG AA Contrast Verification (computed, WCAG 2.x relative luminance)

Paper text-safe variants added to `lib/design/tokens.ts` — every pair ≥4.5:1, including the 14%-alpha chip surfaces the text actually renders on:

| Token | Hex | on #FFFFFF | on #FAF3E9 | on 14% chip/white | on 14% chip/paper |
|---|---|---|---|---|---|
| successText | #1D7355 | 5.78:1 | 5.24:1 | 4.96:1 | 4.54:1 |
| warningText | #A05111 | 5.71:1 | 5.18:1 | 4.95:1 | 4.54:1 |
| dangerText | #B33846 | 5.88:1 | 5.34:1 | 4.95:1 | 4.54:1 |

Raw paper tones (kept for fills only): success 3.35:1, warning 3.00:1, danger 3.87:1 on white — the failure the checker flagged. Ink theme's existing tones serve as its text-safe set: success/warning/danger ≥6.44:1 on all ink grounds (bg/card/cardHi) and ≥5.65:1 on their own 14%-alpha chips; ink.goldText ≥9.08:1.

Additional measured (documented, not changed): paper.ink3 muted 3.96/3.59:1 (why pending uses ink2 at 9.23/8.38:1); paper.goldText 4.64:1 on white cards (pass), 4.21:1 on the paper page bg (place sm/md gold amounts on card surfaces; 22px/800 lg qualifies as WCAG large text where 3:1 applies).

## Decisions Made

- **Darkened the plan's suggested text-safe hexes** (#1F7A5A→#1D7355, #A85512→#A05111, #B93A48→#B33846). The suggested values pass on solid #FFFFFF/#FAF3E9 (4.78–5.58:1) but StatusChip text sits on the 14%-alpha tone blend, where they measure 4.14–4.31:1 over the paper bg — below AA. The plan's own instruction ("verify… darken further if any pair is below 4.5") was applied to the real rendered surfaces.
- **Pending tone foreground = resolved `mutedText`** (paper.ink2 #4A4363, ink.muted) rather than raw paper.ink3, which fails AA at 3.59:1. This satisfies the checker's requirement that penalty/pending foregrounds be text-safe while keeping the de-emphasized look. The optional `sub` line keeps plain muted (ink3) per the plan's explicit spec — it is supplementary metadata, noted here for the verifier.
- **paper.goldText left unchanged** — it is a plan-01 locked token already consumed app-wide via the T re-exports; its 4.21:1 on the page bg (vs 4.64:1 on white cards) is documented above instead of mutated. Changing it is a phase-level design-contract decision, not a plan-02 deviation.
- **Stamp reduced-motion is double-guarded**: CSS (`no-preference` media query around the keyframes — zero animation ever defined for reduce users, no hydration mismatch) + JS (drops the animation property after mount).
