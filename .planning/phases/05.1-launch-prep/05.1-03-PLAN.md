## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm registry → build | Installing @sentry/nextjs pulls third-party code into the build (supply chain) |
| app runtime → Sentry ingest | Error payloads (messages, breadcrumbs, request data) leave the server to a third party |
| public internet → /api/health?boom=1 | Anyone can trigger the test error (a deliberate 500) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-051-SC | Tampering | npm install @sentry/nextjs | mitigate | Legitimacy gate (Task 1, blocking-human) verifies official getsentry publisher on npmjs.com before install |
| T-051-06 | Information Disclosure | Sentry error payloads | mitigate | Do not set sendDefaultPii; request bodies not attached; minimal init — no child names/emails/coins deliberately sent |
| T-051-07 | Information Disclosure | Missing DSN in dev/CI | mitigate | init guarded on DSN presence → no data sent and no crash when unconfigured |
| T-051-08 | Denial of Service | /api/health?boom=1 | accept | Returns a single 500; no state mutation, no amplification; value is worth the prod verification capability |
</threat_model>
