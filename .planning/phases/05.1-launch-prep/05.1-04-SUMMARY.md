## Accomplishments

- `posthog-js@1.396.6` installed — confirmed official PostHog browser SDK (npm publisher `posthog`, repo `PostHog/posthog-js`); checkpoint was pre-approved in-session by the operator
- `lib/analytics.ts` — `'use client'`-safe wrapper exporting `initAnalytics()`, `track()`, `trackPageview()`. `initAnalytics()` only calls `posthog.init` when `NEXT_PUBLIC_POSTHOG_KEY` is truthy and `window` is defined, with `api_host` defaulting to the EU ingestion host `https://eu.i.posthog.com`, `capture_pageview: false` (manual capture, see below), and `disable_session_recording: true`. `track()` no-ops silently (never throws) when the key is absent or init never ran, and additionally wraps the actual `posthog.capture` call in try/catch as defense-in-depth.
- `components/AnalyticsProvider.tsx` — calls `initAnalytics()` once on mount; a `Suspense`-wrapped inner component uses `usePathname()`/`useSearchParams()` to capture a `$pageview` on every client-side route change (the current recommended PostHog App Router recipe, since App Router navigations don't trigger PostHog's own history-based autocapture).
- Mounted `<AnalyticsProvider />` inside `LanguageProvider` in `app/layout.tsx`, alongside `OfflineBanner`/`PushInit`/`InstallPrompt`, as a null-UI sibling. The FamilyCoins metadata export from plan 05.1-02 is untouched.
- Wired `track('day_saved', { role: 'kid' })` in `components/kid/KidDayFillForm.tsx` and `track('day_saved', { role: 'parent' })` in `components/DailyModal.tsx`, both fired only on the post-award success path (`res.ok` branch of the existing `/api/wallet/award` POST), sending no PII — `role` only, no names/emails/childId/familyId/coin amounts.

## User Setup Required

**External service (PostHog) requires manual dashboard configuration and env vars — no code change possible for this part.**

1. Create a PostHog project on **EU Cloud** (free tier) at `eu.posthog.com` → Create project.
2. Copy the project API key from Project Settings → Project API Key.
3. Add to Vercel project env (Production, and Preview/Development as desired):
   - `NEXT_PUBLIC_POSTHOG_KEY` = the project API key
   - `NEXT_PUBLIC_POSTHOG_HOST` = `https://eu.i.posthog.com` (optional — this is already the code default, but setting it explicitly is harmless and self-documenting)
4. Redeploy.
5. **Manual PostHog Live Events confirmation (not yet performed — requires the key to exist first):** after deploy with the key set, visit the site (any page) to generate a pageview, then save a day from either `/kid/day` or a parent daily modal. Within PostHog → Activity → Live Events, confirm a `$pageview` event and a `day_saved` event (with `properties.role` = `kid` or `parent`) both appear. This step is intentionally left for the operator to run after adding the env vars above — the executor has no way to add secrets to Vercel or confirm delivery to a PostHog project it cannot access.
