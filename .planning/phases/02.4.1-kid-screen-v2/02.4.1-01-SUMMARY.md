## What Was Built

### SQL Migration (`supabase-migration-kid-fill-v2.sql`)
Three ALTER TABLE statements ready to run in Supabase SQL Editor:
- `days.filled_by TEXT CHECK (filled_by IN ('child', 'parent')) DEFAULT NULL` — tracks who submitted a day record
- `children.kid_fill_mode INTEGER DEFAULT 1 CHECK (kid_fill_mode IN (1, 2, 3))` — controls what kids can fill in
- `days.mood TEXT DEFAULT NULL` — stores child's emoji mood key (happy/neutral/sad/angry/tired)

### TypeScript Types (`lib/models/child.types.ts`)
- `DayData.filled_by: 'child' | 'parent' | null` added after `homework_done`
- `DayData.mood: string | null` added after `filled_by`
- `Child.kid_fill_mode: 1 | 2 | 3` added after `created_at`

### Repository Updates
- `saveDay()` in `children.repo.ts` now accepts `filledBy?: 'child' | 'parent'` and `mood?: string` params, persists them as `filled_by` and `mood` in the dayData object
- `getSectionsForChildExpenses(childId)` added to `expenses.repo.ts` — returns active sections with id, name, cost, schedule_days for the kid expenses tab
