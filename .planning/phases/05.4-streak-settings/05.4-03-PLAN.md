<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /api/wallet/settings | Parent-authored streak values (untrusted numbers) cross into a money table |
| browser Settings UI → wallet_settings | UI number inputs are convenience only; server is authoritative |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-054-06 | Tampering / Elevation | streak settings write | mitigate | Write goes through the existing `PATCH /api/wallet/settings` (`requireParent` + service-role); `wallet_settings` stays RLS SELECT-only for clients (04.4-03) — the UI adds no new write path |
| T-054-07 | Tampering | cross-family settings write | mitigate | `updateWalletSettings` upserts `id: familyId` / `family_id: familyId` derived server-side from the authenticated parent; client-sent id/family_id are stripped in the settings route |
| T-054-08 | Tampering | absurd values entered in the UI (negative bonus, 0 days) | mitigate | Authoritative clamp in the award route + write-side clamp in the settings route (both from Plan 02); checkpoint step 5 exercises this |
| T-054-SC | Supply chain | no new package installs | accept | UI + i18n only; no dependencies added |
</threat_model>
