## What Was Built

### ScreenHeader (D-13 + D-03)
- Default export, props `{ title, coins, name, avatarUrl?, showLogout? }`.
- Left: Avatar (SVG atom fallback, `<img>` when `avatarUrl` given) + title (Heading role: 18px/700 `base.fontDisplay`, `paper.ink`) + name (Label role, `paper.ink3`).
- Right: `<Amount value={coins} theme="paper" money size="lg"/>` — the only gold in the header.
- `showLogout` renders a 44x44 `paper.accent` (indigo) icon button opening an in-component confirm dialog: title/body/cancel/confirm from `kidNav.logoutConfirm.*`, confirm button filled `paper.dangerText`, cancel neutral. On confirm: `setActiveMemberId(null)` → `supabase.auth.signOut()` → `router.push('/login')` (mirrors KidNav.handleLogout).
- No legacy tokens (`T.coral`/`T.plum`/`T.sun`/Nunito) anywhere in the file; all colors from `lib/design/tokens`.

### Chat read marker (D-04)
- Migration `05.7-01-chat-read-marker.sql`: `ADD COLUMN IF NOT EXISTS chat_last_read_at TIMESTAMPTZ` + idempotent (`DROP POLICY IF EXISTS` → `CREATE`) permissive policy `family_members_self_update` — `FOR UPDATE TO authenticated USING (user_id = auth.uid()) WITH CHECK (user_id = auth.uid())` (mitigates T-057-06).
- `markChatRead(memberId)` — own-row `chat_last_read_at = now()` update via the browser singleton (RLS enforces own-row).
- `getChatUnreadCount(familyId, memberId, lastReadAt)` — `head:true, count:'exact'` on `chat_messages`, `sender_id <> memberId`, `created_at > lastReadAt` when the marker exists (all others' messages when null). Reads stay inside the family's existing SELECT RLS (T-057-07).
