# Phase 5.7 Plan 09: Theme-Aware ChatThread + Paper Kid Chat Summary

Theme-aware ChatThread (paper/ink from lib/design/tokens, Nunito/DM Sans literals deleted) with the kid chat screen on paper, unified under ScreenHeader, marking the thread read on open via markChatRead.

## What Was Built

### Task 1: ChatThread theme-aware (commit b1d8144)
- Replaced the module-level hardcoded `KT` token object (the only literal `"Nunito"`/`"DM Sans"` font remnant in kid-reachable code, SC1) with `resolveChatTokens(theme)`:
  - Grounds (`bg/card/ink/ink2/ink3/line/lineSoft`) map to `paper.*` or `ink.*` from `lib/design/tokens`
  - Fonts map to `base.fontDisplay/fontBody/fontMono` (Bitter/Golos/JetBrains Mono vars)
  - `plum/plumSoft` sourced from `base.indigo/indigoSoft` (values were already identical)
  - `coral/teal/sun` message-role accents kept as ChatThread-local constants (theme-invariant branding, not part of the ground system)
- Added `theme?: 'paper' | 'ink'` prop, default `'paper'` — sole consumer `/kid/chat` renders unchanged; real ink ground values built for 5.8 forward-compat without a consumer (RESEARCH.md Pitfall 4 / A1)
- All sub-components (`ViewModeToggle`, `ChatTabBar`, `DateSep`, `SingleActivity`, `ActivityBurst`, `Avatar`, `ActivityFeed`) take the resolved token object as a `C` prop; `mediaBtnStyle` became a function of `C`
- `ChatPanel.tsx` untouched (parent-side consolidation is 5.8 scope)

### Task 2: Kid chat screen (commit 9f6e00f)
- Mounted shared `ScreenHeader` (D-13) with `t('kidHeader.chat')` title, child name, and gold coin balance (wallet fetched read-only via `getWallet` — RLS SELECT, no money write)
- Renders `<ChatThread theme="paper" .../>`; the page's duplicated header/ViewModeToggle/ChatTabBar markup (129 lines) deleted — ChatThread now runs uncontrolled and renders its own toggle/tab bar
- D-04 mark-read: `markChatRead(memberId)` on open, and again on each incoming realtime message while `document.visibilityState === 'visible'` (own-row RLS write per Plan 03's migration, T-057-16 mitigation holds)
- Realtime subscription and send flow inside ChatThread unchanged
- Page's legacy `T` token import removed; remaining page chrome uses `paper.*`/`base.*`
