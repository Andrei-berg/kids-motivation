# Phase 5.7: kid-redesign - Context

**Gathered:** 2026-07-16
**Status:** Ready for planning

<domain>
## Phase Boundary

The kid UI moves fully onto the unified «family bank» design system: every kid
screen (day, wallet, shop, awards, chat, login) uses the unified tokens/atoms
with no Nunito/coral remnants; navigation consolidates to 5 items with the
leaderboard living as a tab inside Awards; motion is disciplined — coin credit
uses the signature stamp+count-up gesture and confetti fires only for streak
bonuses and level-ups (ROADMAP SC1-SC3).

Visual/structural redesign only: no money logic, award math, or PIN-auth logic
changes. Parent Center redesign is Phase 5.8; legacy pages/globals.css are
Phase 5.11. The user's overarching directive for this phase: maximum quality,
«делаем всё возможное и везде по всему» — apply the frontend-design discipline
(one signature element, restraint elsewhere, copy as design material).

</domain>

<decisions>
## Implementation Decisions

### Navigation (5 items)
- **D-01:** Tab set: **День · Кошелёк · Награды · Магазин · Чат**. Chat is
  promoted from FAB to a first-class 5th tab; the mobile chat FAB is removed.
- **D-02:** Bar style: **full-width «ledger» panel** on paper bg with a hairline
  top border (`paper.line`), replacing the floating glass pill. Active tab is
  **indigo for all tabs** (contract: indigo = interactive) with bold label;
  inactive tabs `ink3`. The legacy per-tab rainbow colors are removed.
- **D-03:** Logout moves off the floating top-right button into the **Day screen
  header** (next to the child's avatar/name). The floating button is deleted.
- **D-04:** Chat tab gets a **real unread counter** backed by a stored last-read
  marker in the DB (new mechanic; schema shape — e.g. `last_read_at` per family
  member vs a read-marks table — is Claude's discretion). Not a money table;
  standard family RLS.

### Awards + leaderboard
- **D-05:** Awards screen = **3 tabs: Бейджи / Стрики / Рейтинг**, with the
  XP bar + level in the shared screen header above the tabs.
- **D-06:** `/kid/leaderboard` route is **deleted without redirect** (404);
  remove all internal links/pushes pointing to it.
- **D-07:** Rating presentation: **podium (top-3 with avatars) + list** — chosen
  «если подиум + список будет круче»; final visual call is Claude's during
  planning/execution, staying within the design contract (gold only on money).
  Must look good with only 2 children.
- **D-08:** Single-child family: the Rating tab becomes **«я против себя»** —
  this week vs last week comparison (needs a prior-week score slice via the
  existing `getWeekScore` aggregation), instead of an empty leaderboard.

### Redesign depth per screen
- **D-09:** `KidDayFillForm` (1417 lines): **reskin + refactor** — split into
  per-block renderer components and re-verse them in the «family bank» language
  (receipt-style cards, ledger price rows). Save/award logic is untouched: only
  render changes. The 5.6 «block type → renderer» contract is the seam.
- **D-10:** `/kid/login` **is included** — first brand impression; visual only,
  PIN/lockout logic untouched.
- **D-11:** Chat: `ChatThread` becomes **theme-aware (paper/ink)** and both
  variants are built in 5.7 — kid side on paper, parent side on ink. User
  decides at verification what stays («сделай оба, что не подойдёт удалим или
  оставим с возможностью выбора»); parent chat must not break, 5.8 finalizes it.
- **D-12:** **All kid-flow overlays are in scope**: GoalsPanel,
  CelebrationOverlay, BadgeNudge, KidChallenges, purchase/exchange modals —
  otherwise SC1 fails (old style pops over new).
- **D-13:** **Unified screen header** («сберкнижка»): one shared component on
  all 5 kid screens — avatar, name, coin balance as gold `Amount`, contextual
  screen title; Day adds the logout button (D-03).
- **D-14:** Shop: **cards + ledger prices** — product card grid stays (kids need
  visual goods), prices via the `Amount` atom (gold, mono), request statuses via
  `StatusChip`, cards restyled to paper tones.
- **D-15:** Desktop: **keep the 4.5 layout structure** (sticky columns,
  sidebar); sidebar nav also becomes the 5-item set in the new style. Only the
  visual language changes.
- **D-16:** Wallet: **hero-«сберкнижка»** — large gold Amount with count-up,
  weekly «+earned / −spent» summary line, goals restyled as «вклады» (deposits)
  with progress toward target; transaction history stays on LedgerRow (5.3
  pilot).

### Motion discipline
- **D-17:** Signature gesture fires **after the server responds**: day save →
  `/api/wallet/award` returns actual coins → stamp «+N 🪙» (450ms
  scale+translateY) + header balance count-up to the new value; awarded items
  render as ledger rows in the day summary. One orchestrated moment; only
  server-confirmed numbers are ever animated. (The client-side `coinsPreview`
  stays an estimate only, never stamped.)
- **D-18:** Ordinary checkbox toggles (room task, activity, book, custom block):
  **micro-tick** — checkmark draw-in + short scale pulse ≤200ms, no particles.
  All current per-toggle confetti calls are removed.
- **D-19:** Confetti triggers detected **from the award response** (server
  already knows it credited a `streak` source) + **level-up by XP threshold
  crossing** computed with the same level formula the Awards screen uses. No new
  DB fields.
- **D-20:** Badges / goals reached / approved purchases & exchanges lose
  confetti and get the **stamp ceremony**: overlay/card with a rubber-stamp
  effect («ПОЛУЧЕНО» / «ЦЕЛЬ ДОСТИГНУТА», −8° tilt, 450ms entrance) — celebratory
  but in the bank aesthetic. `prefers-reduced-motion` respected everywhere
  (Stamp atom already double-guards).

### Claude's Discretion
- Unread-marker schema and read/write path (D-04).
- Final rating layout (podium vs ledger list) if the podium doesn't look
  better in practice (D-07).
- Exact component decomposition of KidDayFillForm blocks (D-09).
- Micro-interaction details of tick/stamp timing curves (D-18/D-20).
- i18n keys for all new/renamed labels (ru/en) following existing dotted-key
  pattern; interface copy follows the frontend-design writing rules (plain
  verbs, sentence case, actions named by outcome).

</decisions>

## Existing Code Insights

### Reusable Assets
- `components/design/atoms.tsx` — theme-aware LedgerRow/Amount/StatusChip/
  Stamp/useCountUp already built and piloted (kid wallet history, parent
  dashboard) — the core kit for every screen here.
- `components/kid/design/atoms.tsx` — AnimatedNum (rAF cubic ease-out),
  Avatar, XPBar, ProgressRing, SectionHeader, CoinPill, KMButton, Confetti —
  migrate the keepers into the unified language, retire the rest.
- 5.6 assembled block renderers inside KidDayFillForm — the reskin decomposes
  along these block boundaries.
- `api.getWeekScore(childId, weekStart)` — already computes weekly scores from
  raw rows; reuse for prior-week «я против себя» comparison (D-08).

### Established Patterns
- Kid tokens already re-export unified palette (5.3): SC1's «no coral
  remnants» means retiring the legacy T key NAMES (coral/sun/teal/pink/mint)
  from kid components, not just values. Nunito remains only in onboarding
  pages — out of 5.7 scope (not kid screens; 5.11 territory).
- Confetti today fires on every checkbox (KidDayFillForm room/activity/book/
  custom toggles), day save, goals, shop purchase, exchange — all these call
  sites change under D-18..D-20.
- i18n via dotted keys in `lib/i18n` (en/ru json); all renamed labels need
  both locales.
- `prefers-reduced-motion` double-guard pattern established in the Stamp atom
  (05.3-02 decision).

### Integration Points
- `middleware.ts` role-guards `/kid/*` — deleting `/kid/leaderboard` needs no
  middleware change, but grep for internal `href`/`router.push` references.
- `/api/wallet/award` response shape — the client reads credited sources from
  it for stamp/confetti triggers (D-17/D-19); if the current response lacks
  per-source detail, extend the response (read-only additive change, no award
  logic change).
- ChatThread is shared with `/parent/chat` — theme prop must default to the
  current parent look so 5.8 can switch deliberately (D-11).
- Unread counter (D-04) touches `chat_messages` reads + a new marker write —
  keep it out of money paths entirely.
