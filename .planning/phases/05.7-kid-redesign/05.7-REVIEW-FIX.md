## Fixed Issues

### CR-01: `family_members_self_update` policy lets a child promote themselves to parent

**Files modified:** `supabase/migrations/05.7-02-chat-read-marker-fix.sql`, `lib/repositories/chat.repo.ts`
**Commit:** b5b74a2 (already fixed prior to this run; verified against current code)
**Applied fix:** Verified — the permissive self-UPDATE policy is dropped and replaced by
a SECURITY DEFINER `mark_chat_read()` RPC (Option 1 from the review) that can touch only
`chat_last_read_at` for `auth.uid()`'s own row, with `REVOKE ALL FROM PUBLIC` +
`GRANT EXECUTE TO authenticated`. `markChatRead()` in `chat.repo.ts` now calls
`supabase.rpc('mark_chat_read')` instead of `.update(...)`. No re-application needed.

### WR-01: Three of four wallet quick actions are dead — one navigates to a non-existent route

**Files modified:** `app/kid/wallet/page.tsx`
**Commit:** 59637e2
**Applied fix:** "Stats" now points at `/kid/achievements` (existing screen with the rating
tab) instead of the non-existent `/kid/analytics`; the "gift" action was dropped (no P2P
entry point exists yet) and the grid reduced to 3 columns; `id="goals"` was added to the
GoalsPanel wrapper and hash hrefs now scroll in-page via `scrollIntoView` (a `router.push`
to the same path does not reliably re-trigger hash scrolling in the App Router).

### WR-02: `normalizeDate(new Date())` used for "today" — the exact UTC-date bug CLAUDE.md forbids

**Files modified:** `app/kid/wallet/page.tsx`, `app/kid/achievements/page.tsx`
**Commit:** f138115
**Applied fix:** All three "today" call sites (wallet lines 58/89, achievements line 135)
now use `localDateString()` from `utils/helpers`; the now-unused `normalizeDate` imports
were replaced. `addDays(today, -7)` downstream accepts the string form unchanged.

### WR-03: Exception after a successful award silently strands the save — and invites duplicate grade rows

**Files modified:** `components/kid/KidDayFillForm.tsx`
**Commit:** e0f43ca
**Applied fix:** `checkAndAwardBadges(childId, date)` is now wrapped in its own
try/catch (non-fatal warn) so that once `awardOk` is true, `onSaved({...})` is always
reached — a badge failure can no longer leave the form in edit mode and provoke a
duplicate-row re-submit. `handleRetryAward` gained a catch that logs and sets
`saveError(true)` instead of producing an unhandled rejection.

### WR-04: `getMessages` loads the OLDEST 50 messages — new messages become unreachable

**Files modified:** `lib/repositories/chat.repo.ts`
**Commit:** fb82cea
**Applied fix:** `getMessages` now orders `created_at` descending with `limit(limit)` and
returns `(data ?? []).reverse()`, so the chat always shows the latest N messages in
chronological order — consistent with the unread badge introduced this phase.

### WR-05: ChatThread hardcodes Russian UI strings despite full i18n elsewhere in the phase

**Files modified:** `components/chat/ChatThread.tsx`, `public/locales/en.json`, `public/locales/ru.json`
**Commit:** b0f6f86
**Applied fix:** Added `chat.viewMixed/viewSplit/tabMessages/tabActivity/noActivity/`
`newBadge/today/yesterday/eventsOne/eventsFew/eventsMany` keys to both locale files and
keyed every flagged literal (`Лента`, `Раздельно`, `💬 Сообщения`, `✨ Активность`,
`Нет активности`, `Новый значок!`, `Сегодня`/`Вчера`, event counts). The event-count
plural now uses correct Russian rules (`1/21/31 → событие`, `2–4/22–24 → события`,
else `событий`) via an `eventsCountKey()` selector that also maps correctly onto the EN
one/few/many keys. `formatDate`/`processMessages`/`buildDayGroups` take the `t` callback;
`ViewModeToggle`, `ChatTabBar`, `SingleActivity`, `ActivityBurst`, and `ActivityFeed`
now call `useT()`.

### WR-06: Shop approval sheet stamps «ПОЛУЧЕНО» on a timer, not on a real state

**Files modified:** `app/kid/shop/page.tsx`, `public/locales/en.json`, `public/locales/ru.json`
**Commit:** 8fc589f
**Applied fix:** The timer-driven stage in `ApprovalSheet` now stamps the new
`stamp.requestSent` («ЗАЯВКА ОТПРАВЛЕНА» / "Request sent") — which IS server-confirmed at
that point (requestPurchase succeeded, coins deducted) — instead of `stamp.received`.
The real «ПОЛУЧЕНО» stamp is now rendered in the "Мои заявки" purchases list only when
`p.status === 'approved' || 'delivered'`, replacing the StatusChip for those rows;
pending/rejected rows keep their StatusChip.
