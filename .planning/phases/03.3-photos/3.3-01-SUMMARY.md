## Decisions Made

- No unit tests for `compressImage` — canvas APIs require heavy DOM mocking in Node; TypeScript type-check is the verification gate (plan explicitly calls this out)
- Files under 100KB skip compression to avoid unnecessary canvas round-trips for already-small photos
- `upsert: true` in `uploadPhoto` allows callers to overwrite proofs at the same path (e.g., re-submission)
- Bucket is private (public=false); all read access goes through signed URLs — prevents public enumeration of family photos
