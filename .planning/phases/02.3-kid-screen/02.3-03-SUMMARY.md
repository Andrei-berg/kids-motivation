## What Was Built

### /kid/wallet page (app/kid/wallet/page.tsx — 419 lines)

**Balance Hero:**
- `kid-coin-gradient` card with `kid-coin-bounce` emoji animation
- Large coin count (text-5xl font-extrabold), money balance, earned/spent pills (bg-white/20)
- All from `getWallet(activeMemberId)`

**Quick Actions (2 buttons):**
- "Запросить вывод" (violet) — disabled when money balance = 0; opens withdrawal bottom sheet
- "Обменять монеты" (amber) — toggles inline tooltip "Попроси родителя обменять монеты в личном кабинете"

**Weekly Earnings (This Week):**
- 4 rows with color dots: Оценки (blue), Комната (emerald), Поведение (violet), Спорт (orange)
- Data from `api.getWeekScore(activeMemberId, weekStart)` — real fields mapped (coinsFromGrades, coinsFromRoom, coinsFromBehavior)
- Total row at bottom

**Transaction History:**
- Groups transactions by date using `created_at.slice(0, 10)` key
- Sticky date headers with relative labels (Сегодня / Вчера / formatted date)
- Each row: circular icon + description + colored coins_change (+/-)
- Empty state: piggy bank emoji + message
- "Загрузить ещё" button: grows limit by 20 on each click (20→40→60...)
- `hasMoreTx` hides button when fetched count < limit

**My Requests (conditional):**
- Purchase rows: icon + reward_title + price + fulfilled-based status badge
- Withdrawal rows: icon + amount + status badge from CashWithdrawal.status
- Badge colors: Ожидает=amber, Одобрено=emerald, Отклонено=rose

**Withdrawal Bottom Sheet:**
- Fixed overlay with `kid-bottom-sheet` class, stop-propagation on inner panel
- − / + buttons adjust amount in steps of 10 (min 1)
- Submits via `requestWithdrawal(activeMemberId, withdrawAmount)`, reloads withdrawals
- Shows success toast for 3 seconds after submission

**Loading State:**
- 3 skeleton placeholders (`kid-skeleton`) while data loads

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] getTransactions signature mismatch — no offset parameter**
- **Found during:** Task 1
- **Issue:** Plan spec showed `getTransactions(childId, limit, offset)` but actual implementation only accepts `(childId?, limit?)` — no offset/cursor pagination
- **Fix:** Implemented load-more via growing limit: start at 20, add 20 per click. `hasMoreTx` = `fetched.length === currentLimit`
- **Files modified:** app/kid/wallet/page.tsx

**2. [Rule 1 - Bug] api.getWeekScore returns different field names than plan**
- **Found during:** Task 1 (TypeScript error)
- **Issue:** Plan specified `{ grades, room, behavior, sport }` but actual service returns `{ coinsFromGrades, coinsFromRoom, coinsFromBehavior, total, gradedDays, roomOkDays, filledDays }` — no sport coins calculated
- **Fix:** Updated weekScore state type and rendering to use actual field names; sport row displays 0 (service doesn't calculate sport coins from section data)
- **Files modified:** app/kid/wallet/page.tsx

**3. [Rule 1 - Bug] RewardPurchase has no status field**
- **Found during:** Task 1
- **Issue:** Plan referenced `status` on RewardPurchase but type only has `fulfilled: boolean`
- **Fix:** Mapped `fulfilled: true` → 'approved', `fulfilled: false` → 'pending' for badge display
- **Files modified:** app/kid/wallet/page.tsx
