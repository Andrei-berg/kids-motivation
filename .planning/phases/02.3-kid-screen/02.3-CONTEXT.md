# Phase 2.3: kid-screen - Context

**Gathered:** 2026-04-05
**Status:** Ready for planning

<domain>
## Phase Boundary

Build `/kid/*` routes — a bright, gamified read-only experience for children. Kids see today's tasks, wallet balance + history, achievements (badges + streaks + XP), and can browse the shop and send purchase requests to parents.

Parent enters all data. Kids are read-only. Shop purchases require parent approval (Phase 2.4 handles approval flow). Leaderboard is the only place where a child can see sibling data.

</domain>

<decisions>
## Implementation Decisions

### Visual Theme
- Bright white / light background — opposite of parent dark theme
- Colorful gradient accents per section: green for Day, gold for Wallet, purple for Achievements, blue for Shop
- Inspiration: Duolingo + ClassDojo + GoHenry
- Larger font than parent UI, generous padding, fewer items per screen — mobile-first
- Each kid page has a persistent header: child emoji + name + current coin balance
- Animations: CSS-only, subtle — fade-in on load, progress bar fills, number counters. No heavy libraries.
- Full-screen celebration overlay reserved for badge unlock moments only

### Navigation
- Bottom tab bar with 5 tabs: Day | Wallet | Achievements | Shop | Leaderboard
- App redirects `/kid` → `/kid/day` (no separate hub/landing page)
- Kid screen is locked to the logged-in child — no child switcher
- Siblings visible only in Leaderboard tab
- Each tab is a separate Next.js route under `/kid/*`

### Badge Showcase + New Badges
- **Existing 6 badges:** `week_excellent` 🌟, `clean_master` 🧹, `sportsman` 💪, `goal_achiever` 🎯, `perfect_week` 👑, `study_lover` 📚
- **4 new badges to add:**
  - `coin_saver` 💰 — accumulate N coins without spending (rewards saving habit)
  - `first_purchase` 🛒 — first approved shop purchase
  - `streak_30` 🔥 — any streak reaches 30 days
  - `full_week_grades` 📋 — grades entered for all scheduled subjects in a week
- **Locked badges:** greyed-out silhouette + lock icon — shape visible but monochrome; mystery preserved
- **Progress toward badges:** show progress bar per badge (e.g. "Sportsman: 9/14 days") — motivates kids
- **Badge earned celebration:** full-screen overlay with big badge + name + XP awarded, tap to dismiss; triggered on next page load after condition is met

### Shop Browse
- 2-column card grid: big emoji icon + item name + coin price per card
- Unaffordable items: visible but greyed out + "Need N more coins" label — aspirational, not hidden
- Buy flow: tap item → confirm modal (item + price + "Send to parent?") → tap Confirm → toast "Request sent! 📨"
- Pending requests visible: "My Requests" section in Wallet tab (or Shop tab) showing status Pending ⏳ / Approved ✅ / Rejected ❌

### Maintainability (key principle)
- Badge definitions as a single data array — add new badge = add one object, no logic changes
- Kid theme CSS variables in a separate `kid-theme.css` or scoped CSS block — easy to tweak colors
- Each kid tab is an independent page component — no shared state between tabs except header
- Reuse `getRewards()`, `getWallet()`, `getTransactions()`, `api.getStreaks()` from existing repos — no new API surface

</decisions>
