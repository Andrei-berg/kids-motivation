**Change 2: Fix handleJoinAsChild() — use child_id from family_members after claim**
Replace the current `handleJoinAsChild` implementation with:
```typescript
const handleJoinAsChild = async () => {
  if (!selectedChildId || !userId) return
  const child = unlinkedChildren.find((c) => c.memberId === selectedChildId)
  if (!child) return
  setLoading(true)
  setError(null)
  try {
    await joinFamilyAsChild(familyId, userId, { memberId: child.memberId, displayName: child.displayName })

    // After claim, read child_id from the now-linked family_members row
    const supabase = createClient()
    const { data: claimedRow } = await supabase
      .from('family_members')
      .select('child_id')
      .eq('user_id', userId)
      .eq('family_id', familyId)
      .maybeSingle()

    setStoreFamilyId(familyId)
    if (claimedRow?.child_id) {
      setActiveMemberId(claimedRow.child_id)
    }
    router.push('/kid/day')  // child lands on kid screen
  } catch (err) {
    setError(err instanceof Error ? err.message : 'Не удалось присоединиться. Попробуйте снова.')
  } finally {
    setLoading(false)
  }
}
```

**Change 3: Fix handleJoinAsAdult() — redirect to /parent/dashboard**
Replace `router.push('/dashboard')` with `router.push('/parent/dashboard')` in `handleJoinAsAdult`.

**Change 4: Fix the "already has family" early-exit in useEffect**
In the existing auth init useEffect, when an existing family_members row is found (lines ~696–704), change the redirect from `router.replace('/dashboard')` to role-based:
```typescript
if (existing) {
```
