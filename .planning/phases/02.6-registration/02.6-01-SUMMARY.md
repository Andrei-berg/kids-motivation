## What Was Built

### `generateChildId(name: string): string` (internal, not exported)

- Lowercases name, replaces non-alphanumeric with `_`, collapses repeated underscores, strips leading/trailing underscores
- Appends 4-char random alphanumeric suffix
- Result always matches `/^[a-z0-9][a-z0-9_]{1,18}[a-z0-9]$/` (3–20 chars, no leading/trailing underscores)

### `createChildWithWallet(familyId, child: ChildInput): Promise<ChildCreateResult>` (exported)

- Generates child ID via `generateChildId(child.name)`
- Inserts row in `children`: `{ id, name, family_id, active=true, xp=0, level=1, kid_fill_mode=1 }`
- On `23505` unique violation (id collision): retries once with a new ID
- Inserts row in `family_members`: `{ family_id, user_id=null, role='child', display_name, child_id }`
- Inserts row in `wallets`: `{ child_id, family_id, coins=0, total_earned=0 }`
- Returns `{ childId: string, memberId: string }`

### `completeOnboarding(familyId, coinRules?): Promise<void>` (exported)

- Upserts `wallet_settings` with `id='default'`
- Defaults: grade 5→+5, grade 4→+3, grade 3→**-3**, grade 2→**-5** (negative penalties per CLAUDE.md)
- Room: +3, good behavior: +5, exercise: +5
- Coach: 5→+10, 4→+5, 3→0, 2→+3 (stored positive, applied as deduction), 1→+10 (deduction)
- `coinRules` param allows overriding `g5`, `g4`, `g3`, `g2`, `room`, `sport`
- P2P limits: transfer=100, day=200, month=500, debt=200; exchange rate=10
- Idempotent (upsert handles repeated calls)

### Exported Types

- `ChildInput { name: string }`
- `ChildCreateResult { childId: string; memberId: string }`
