<task type="auto">
  <name>Task 2: InstallPrompt component for Android + iOS</name>
  <files>components/InstallPrompt.tsx, app/layout.tsx</files>
  <action>
Create components/InstallPrompt.tsx as a 'use client' component:

**Android Chrome:** Listen for `beforeinstallprompt` event on window. Store the event in a ref. Show a dismissible bottom banner with "Добавить на главный экран" button. On click, call `event.prompt()`. On dismiss, save `pwa_install_dismissed` to localStorage and hide banner permanently.

**iOS Safari:** Detect iOS via `/iPad|iPhone|iPod/.test(navigator.userAgent)` AND check NOT already standalone via `window.matchMedia('(display-mode: standalone)').matches`. If iOS + not standalone: show a different dismissible bottom banner with instructions: "Нажмите [иконка поделиться] → «На экран "Домой"»". Dismiss saves `pwa_ios_dismissed` to localStorage.

**Both banners:** Only show if not previously dismissed (check localStorage on mount). Render as a fixed bottom bar (z-index: 1000) above everything else. Style with dark background (`#1e1b4b`), white text, indigo accent button. Include an X close button.

**Do NOT show:** If already in standalone mode (`window.matchMedia('(display-mode: standalone)').matches` is true) — hide both banners.

Mount in app/layout.tsx alongside PushInit:
```tsx
import { InstallPrompt } from '@/components/InstallPrompt'
// In body:
<PushInit />
<InstallPrompt />
{children}
```

Use named export `export function InstallPrompt()`.

The component should handle SSR safely — all window/navigator checks inside useEffect or with `typeof window !== 'undefined'` guards.
