# Phase 05.11 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope Note (read first)

This phase is **not** a UI-building phase. Per `05.11-CONTEXT.md`, it: (1) deletes 10 legacy
page routes and replaces them with server-level `next.config.js` redirects, (2) shrinks
`globals.css` to remove now-dead classes, and (3) ships the FamilyCoins app-icon artwork +
fixes stale manifest/viewport color fields. No new screen, component, or user flow is being
designed. Accordingly:

- **Redirects (D-01 through D-04):** No visual/interaction contract applies — a
  `next.config.js` redirect has no rendered UI. The pages a user actually lands on
  (`/parent-center`, `/parent/settings`) already have their own locked UI-SPECs from
  Phases 5.7/5.8. Nothing to add here.
- **globals.css purge:** Pure deletion of dead CSS classes that only the removed legacy
  pages referenced. It must not change the rendered output of any surviving page — there
  is no new visual contract to write; the contract is "byte-identical rendering of every
  surviving page," verified by build + visual pass (already specified as Claude's
  discretion / verification method in CONTEXT.md).
- **Manifest/viewport color fields (D-08):** Values are already locked by CONTEXT.md
  against `lib/design/tokens.ts` (`theme_color: #6C5CE7`, `background_color: #141021`,
  `start_url: /`). Recorded below for completeness; not re-litigated.
- **App icon / brand mark (D-05 through D-07):** The one genuinely new visual artifact
  this phase produces. Fully specified below — this is the section the checker and
  executor should treat as the real design contract for this phase.

## App Icon / Brand Mark — Full Visual Spec

The one new visual deliverable in this phase. Concept, colors, and "not a monogram / not a
mini ledger-row" constraint are locked by CONTEXT.md D-05/D-06/D-07. Everything below fills
in the buildable detail CONTEXT.md left open.

### Concept
A flat, two-tone **coin medallion** mark: a gold coin centered on the indigo brand-square
background. No wordmark, no drop shadows, no gradients or bevels beyond a single flat rim
ring — matches the flat-token aesthetic of the rest of the design system (no gradients
exist anywhere in `lib/design/tokens.ts`).

### Inner detail: star (resolved — was left either/or in CONTEXT.md)
Use a **5-point star**, not a checkmark. Rationale (Claude's discretion, per CONTEXT.md's
explicit delegation of this choice): the app already uses a checkmark glyph as the
task-complete / Tick atom throughout the kid UI (`components/kid/design/` Tick component,
checklist items). Reusing a check on the app icon would visually conflate "brand mark" with
"task done" state at a glance. A star reads unambiguously as "reward/coin" and stays legible
at 16-24px favicon-equivalent scale (a check's diagonal strokes anti-alias poorly at very
small sizes; a symmetric star holds its silhouette better).

### Canvas and layers (single source artwork, per D-07)
Design at **512×512px**, export down for smaller sizes (do not redesign per size).

| Layer | Shape | Size (% of 512 canvas) | Color | Notes |
|-------|-------|------------------------|-------|-------|
| 1. Background | Full-bleed square, no rounded corners baked in | 512×512 (100%) | `#6C5CE7` | OS applies its own mask (circle/squircle/rounded-square); do not pre-round |
| 2. Coin disc | Circle, centered | 66% diameter (≈338px) | `#E9A83C` fill | Centered at (256, 256) |
| 3. Coin rim | Ring stroke on the coin disc's outer edge | stroke width 3% of canvas (≈15px), drawn just inside the disc edge | `#C6871E` | Flat stroke only — no gradient/bevel |
| 4. Star cutout | 5-point star, centered inside the coin | 45% of the coin disc diameter (≈152px) | `#6C5CE7` (same as background — the star is negative space, not a third color) | Keeps the mark to exactly 2 brand hues (indigo + gold), consistent with "gold = money only" |

Result: an indigo square with a gold coin in the middle, and an indigo star cut out of the
coin's center (the background color shows through). No third color anywhere in the mark.

### Maskable safe zone (critical — governs the 66% choice above)
`manifest.ts` currently declares `purpose: 'maskable'` on the 192px icon and on one of the
two 512px entries. Maskable icons must keep all meaningful content inside the **inner 80%
"safe zone"** circle (per the Web App Manifest maskable-icon spec) — anything outside that
gets clipped by adaptive-icon masks on Android. The 66%-diameter coin disc (plus its rim)
sits well inside that 80% safe zone with margin to spare; do not enlarge the coin past ~72%
diameter or it risks clipping on aggressive OS masks.

### Export matrix
Regenerate all three existing files from this one artwork — same design, different raster
sizes only, per D-07:

| File | Size | Purpose | Format notes |
|------|------|---------|--------------|
| `public/icon-192x192.png` | 192×192 | `maskable` (per current `manifest.ts` entry) | PNG, sRGB, flattened (no alpha channel — background is fully opaque indigo, so alpha serves no purpose and can cause edge artifacts on some Android launchers) |
| `public/icon-512x512.png` | 512×512 | used twice in `manifest.ts`: once `purpose: 'any'`, once `purpose: 'maskable'` | Same flattened PNG serves both entries — no separate art needed since the safe-zone constraint above already satisfies both purposes |
| `public/apple-touch-icon.png` | 180×180 (Apple's standard touch-icon size) | iOS home screen | PNG, sRGB, **must be fully opaque with no alpha channel** — iOS requirement; iOS applies its own corner rounding, so do not pre-round or add padding beyond what's already in the 66%/80% layout above |

Do not introduce a `favicon.ico` or any 4th icon file — out of scope per CONTEXT.md (only
the 3 named files are listed for regeneration).

### What NOT to do (explicitly ruled out by CONTEXT.md)
- No monogram / letter "F" mark.
- No miniature `LedgerRow`/ledger-row-style mark.
- No gradient, photorealistic bevel, drop shadow, or texture — flat two-tone only.
- No text/wordmark baked into the icon.
- Do not design per-size (192 vs 512 vs apple-touch) — one 512px master, scaled down.
