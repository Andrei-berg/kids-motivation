# Phase 05.11: legacy-cleanup - Context

**Gathered:** 2026-07-23
**Status:** Ready for planning

<domain>
## Phase Boundary

One UI remains; the brand is consistent everywhere. Concretely:
1. The 10 legacy routes (`/dashboard`, `/wallet`, `/analytics`, `/wallboard`,
   `/expenses`, `/settings`, `/streaks`, `/records`, `/audit`, `/coach-rating`)
   stop having their own page code — they become real HTTP redirects into the
   kid/parent-center equivalents, and the page files are deleted.
2. `globals.css` shrinks to reset + genuinely shared utilities — no classes
   left that only the deleted legacy pages used.
3. App icon, splash, and manifest carry real FamilyCoins branding (name/title
   are already "FamilyCoins" — what's missing is the actual icon artwork and
   stale manifest fields).

Out of scope: restyling any surviving page, touching `/parent/*` desktop
stub routes (not on the deletion list), building a replacement for wallboard,
or any schema/server change.

</domain>

<decisions>
## Implementation Decisions

### Wallboard removal
- **D-01:** `/wallboard` (617-line TV/kiosk display of family wallets, goals,
  badges — the one legacy route with real, unique functionality; nothing
  else in the app replicates it) is cut with **no replacement and no deferred
  backlog note**. It redirects like the other 9 legacy routes. The user
  explicitly chose "cut it, no replacement" over "note it as a deferred
  idea" — do not add a backlog/memory entry for a future TV-mode feature.
- **D-02:** `components/NavBar.tsx` is used ONLY by `app/wallboard/page.tsx`
  (verified via grep — no other file imports it). Once wallboard's page is
  deleted, NavBar.tsx has zero remaining consumers and should be deleted too.

### Extended-role / fallback landing target
- **D-03:** Two places currently hard-redirect to `/dashboard` and must be
  repointed to **`/parent-center`** now that `/dashboard` is deleted:
  - `middleware.ts:74` — the `else` branch of the role-based landing switch
    at `pathname === '/'` (covers `extended` role and any unknown/missing
    role case).
  - `app/page.tsx:37` (`postAuthPath` helper) and `app/page.tsx:44` (the
    auto-redirect-if-already-logged-in effect) — both currently do
    `hasFamily ? '/dashboard' : '/onboarding'` / `router.replace(... : '/dashboard')`.
  These should point at `/parent-center` directly (not `/parent/dashboard`,
  which is itself just a redirect stub) — extended members get the same hub
  a parent sees; no new extended-specific view is being built in this phase.

### Redirect mechanism (Claude's discretion — no user preference either way)
- **D-04:** The 9 non-wallboard legacy routes are *already* 2-line
  `redirect()` page stubs (confirmed via direct read of each file), each
  pointing at a `/parent/*` stub that itself redirects again into
  `/parent-center` (e.g. `/dashboard` → `/parent/dashboard` → `/parent-center`,
  a double hop). Convert these to real redirects in `next.config.js`
  `redirects()` (permanent, server-level — no compiled route left at all) and
  delete the page files/directories. `/parent/*` stub routes are NOT on the
  deletion list and stay as-is — do not delete or restyle them, just leave
  the legacy routes' `next.config.js` entries pointing at whatever the
  current stub already resolves to (verify each target still exists before
  finalizing the redirect map — see per-route targets in Code Context below).

### App icon / brand mark
- **D-05:** Current icon assets (`public/icon-192x192.png`,
  `icon-512x512.png`, `apple-touch-icon.png`) are all an identical **flat
  indigo square with no logo, mark, or text** — verified by viewing the
  files. This phase designs and ships an actual mark, not just a color swap.
- **D-06:** Mark concept: a **coin/medallion mark** — a simple stylized coin
  (circle + inner detail, e.g. a star or check) rendered in coin-gold
  (`#E9A83C`) on the indigo (`#6C5CE7`) brand-square background. Ties directly
  to the "earn coins" core mechanic and to the existing design-system rule
  that gold is reserved exclusively for money/reward elements. Not a
  monogram ("F") and not a miniature ledger-row — those options were
  explicitly passed over.
- **D-07:** Regenerate all icon sizes (192, 512, apple-touch-icon) from this
  one mark design — same artwork, not separate designs per size.

### Manifest metadata refresh
- **D-08:** `app/manifest.ts` has two stale fields to fix in this phase:
  `start_url: '/dashboard'` (a route being deleted — must point at a route
  that still exists after cleanup, e.g. `/` which resolves correctly via
  middleware's role-based landing) and `theme_color`/`background_color`
  (currently `#6366f1`/`#0f0f23`, pre-dating the locked design tokens —
  update to the locked palette `#6C5CE7` indigo / `#141021` ink to match
  `lib/design/tokens.ts`). `app/layout.tsx`'s `viewport.themeColor` (currently
  also `#6366f1`) should be updated to match.

### Claude's Discretion
- Exact grep/verification methodology for the `globals.css` purge (which
  classes are genuinely dead vs. still referenced by a surviving page) is an
  implementation detail, not a product decision — no user preference was
  expressed. Verify via full-repo grep for each class name before deleting,
  and confirm with `npm run build` + a visual pass of surviving pages
  afterward.
- Whether the redirect map lives as one `redirects()` array or is grouped/
  commented by legacy route — implementation detail.

</decisions>
