**Analog:** itself — the surrounding `if/else if` chain already establishes the pattern.

**Current code** (`middleware.ts:63-77`):
```typescript
// Role-based post-login redirect at root / (REQ-ROLE-001)
if (pathname === '/') {
  const url = request.nextUrl.clone()
  if (!membership) {
    url.pathname = '/onboarding'
  } else if (membership.role === 'parent') {
    url.pathname = isMobileUA(request) ? '/parent-center' : '/parent/dashboard'

  return NextResponse.redirect(url)
}
```

## Shared Patterns

### Redirect-stub pattern (being retired, not propagated)
**Source:** every one of the 9 non-wallboard legacy routes, e.g. `app/dashboard/page.tsx`:
```typescript
import { redirect } from 'next/navigation'
export default function Page() { redirect('/parent/dashboard') }
```
This is the pattern being *eliminated*, replaced by `next.config.js` `redirects()` entries
(see above). Do not create new files using this stub pattern — it's precisely what this
phase removes. Listed here only so the executor recognizes it on sight when deleting.

### Design-token sourcing for color literals
**Source:** `lib/design/tokens.ts` (`base.indigo = '#6C5CE7'`, `ink.bg = '#141021'`)
**Apply to:** `app/manifest.ts` (`theme_color`, `background_color`) and `app/layout.tsx`
(`viewport.themeColor`). Both files currently hard-code hex literals rather than importing
from `tokens.ts` — this phase's fix keeps that same hard-coded-literal convention (manifest
and layout are config-level files outside the component tree that normally consumes
`components/kid/design/tokens.ts` / `components/parent-center/tokens.ts` re-exports), just
updates the literal values to match. Do not introduce a new import of `lib/design/tokens.ts`
into these two files unless the planner decides that's preferable — CONTEXT.md only asks for
matching values, not a refactor to import the source module.
