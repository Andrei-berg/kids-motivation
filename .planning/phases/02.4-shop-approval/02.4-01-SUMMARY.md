## Decisions Made

- Existing rows default to `status='approved'` via SQL DEFAULT: backward compat without data migration
- `auto_approve=true` triggers immediate deduction (status='approved', processed_by='auto') — matches old purchaseReward() behavior
- `frozen_coins` stored on the purchase row (not computed from wallet) for clarity and auditability
- `rejectPurchase` requires no wallet update because coins were never deducted in the pending flow
