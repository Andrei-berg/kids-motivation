<task type="auto">
  <name>Task 2: New purchase lifecycle repository functions</name>
  <files>lib/repositories/wallet.repo.ts</files>
  <action>
    Add five new exported functions to `lib/repositories/wallet.repo.ts` after the existing `getPurchases` function. Do NOT modify or remove existing functions — add below them.

    **`createPurchaseRequest(childId, rewardId)`**
    - Fetch reward by rewardId. Throw if not found or not active.
    - Fetch wallet for childId. Throw if not found.
    - Check coins: if reward.reward_type === 'coins', wallet.coins must be >= reward.price_coins. Throw 'Insufficient coins' if not.
    - Check if reward has auto_approve === true. If yes: deduct coins immediately (call updateWalletCoins), insert reward_purchases row with status='approved', return purchase.
    - Otherwise (normal flow): do NOT deduct coins. Insert reward_purchases row with:
      - status: 'pending'
      - frozen_coins: reward.price_coins (or 0 if money type)
      - fulfilled: false
      - All reward title/icon/type/price fields copied from reward
      - balance_after_coins: wallet.coins (unchanged — not deducted yet)
      - balance_after_money: wallet.money
    - Also increment reward.purchase_count by 1.
    - Return the inserted RewardPurchase.

    **`approvePurchase(purchaseId, processedBy?)`**
    - Fetch the purchase. Throw if not found or status !== 'pending'.
    - Fetch the wallet for purchase.child_id.
    - If reward_type === 'coins': deduct purchase.frozen_coins from wallet.coins (call updateWalletCoins with negative amount and description "Покупка одобрена: {reward_title}", icon purchase.reward_icon).
    - Update reward_purchases: set status='approved', processed_by=processedBy||'parent', processed_at=now().
    - Return updated purchase.

    **`rejectPurchase(purchaseId, rejectionNote?, processedBy?)`**
    - Fetch the purchase. Throw if not found or status !== 'pending'.
    - No wallet change needed (coins were never deducted).
    - Update reward_purchases: set status='rejected', rejection_note=rejectionNote, processed_by=processedBy||'parent', processed_at=now().
    - Return updated purchase.

    **`deliverPurchase(purchaseId, note?, processedBy?)`**
    - Fetch the purchase. Throw if not found or status !== 'approved'.
    - Update reward_purchases: set status='delivered', fulfilled=true, fulfilled_at=now(), fulfilled_note=note, processed_by=processedBy||'parent'.
    - Return updated purchase.

    **`getPendingPurchases(childId?)`**
    - Query reward_purchases WHERE status='pending', order by purchased_at ASC (oldest first).
    - If childId provided, filter by child_id.
    - Return RewardPurchase[].
  </action>
