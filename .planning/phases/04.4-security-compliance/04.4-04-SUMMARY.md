# Phase 4.4 Plan 04: Data Export & Account Deletion Summary

**One-liner:** ZIP data export + cascade account deletion API routes with Settings Account tab — Danger Zone pattern, data summary, download-first nudge, and type-to-confirm DELETE friction.

## What Was Built

### Task 1: Export and Delete-Account API Routes

**GET /api/export**
Fetches all family data (children, wallet_transactions, subject_grades, chat_messages, expenses, days, badges), builds a ZIP containing `family-data.json` and per-table CSV files, and returns it as `application/zip` with `Content-Disposition: attachment`.

**DELETE /api/delete-account**
Authenticates the session, verifies parent role, cascades deletion in FK-safe order: chat_messages → per-child data (wallet_transactions, wallet, subject_grades, days, home_sports, streaks, reward_purchases, badges, expenses) → children → family_members → families → `admin.deleteUser(userId)`.

**GET /api/family-summary**
Lightweight count endpoint returning `{ children, transactions, grades }` for the Danger Zone data summary display.

All three routes use `createClient()` from `@/lib/supabase/server` for session auth.

**jszip** installed as a new dependency for ZIP generation.

### Task 2: Settings Account Tab

Added `AccountTab` component to `components/parent-center/screens/Settings.tsx` with:
- **Export section**: "Download Data Export" button that calls `/api/export`, triggers blob download, and calls `insertAuditEvent({ action_type: 'data_export' })`.
- **Danger Zone** (red-bordered): data summary (children/transactions/grades counts from `/api/family-summary`), prominent "download your data first" link above the type-to-confirm input, exact `DELETE` match required to enable the button.
- Account tab added to the tabs array with `t('account.tabLabel')` for i18n.
- `useT()` hook added to `SettingsScreen` component.

**i18n keys** added to both `public/locales/en.json` and `public/locales/ru.json` under the `"account"` section (28 keys total).

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] JSZip arraybuffer instead of nodebuffer**
- **Found during:** Task 1
- **Issue:** `nodebuffer` (and `uint8array`) output types are not assignable to Next.js `Response` body (`BodyInit`). TypeScript error TS2345.
- **Fix:** Changed to `type: 'arraybuffer'` which satisfies `BodyInit` in Next.js Edge runtime.
- **Files modified:** app/api/export/route.ts
- **Commit:** d7d3a04
