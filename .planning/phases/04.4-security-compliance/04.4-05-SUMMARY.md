# Phase 04.4 Plan 05: COPPA/GDPR Consent Gate Summary

COPPA/GDPR parental consent modal in FamilyManager — blocks child creation under 13 until parent explicitly checks consent checkbox; consent_given written to family_members.

## What Was Built

**ConsentGateModal** (inline in `FamilyManager.tsx`):
- Full-screen overlay (position: fixed, z-index: 1000) with centered card
- Lists 4 data categories collected for the child
- Checkbox: "I consent as the parent/guardian to data collection for this child"
- Confirm button (disabled until checkbox checked) calls `doCreateChild(true)`
- Cancel button closes modal without creating the child

**handleAddChild flow change:**
- Validates name and familyId/userId first
- Computes `age = currentYear - birthYear`
- If `age < 13`: sets `showConsentGate = true`, returns early (no child created)
- If `age >= 13`: calls `doCreateChild(null)` directly (no gate, consent_given = null)

**doCreateChild(consentGiven: boolean | null)**:
- Extracted from old `handleAddChild` body
- Handles avatar upload, family_members insert, state update, form reset
- Passes `consent_given: consentGiven` to the Supabase insert
- Resets `showConsentGate` and `consentChecked` on success

**onboarding-api.ts `addChildToFamily`:**
- Added optional 4th param `consentGiven?: boolean | null`
- Passes `consent_given: consentGiven ?? null` to the `family_members` insert

**i18n:**
- `en.json` and `ru.json` both have `consent.*` section with 9 keys (title, subtitle, dataListTitle, dataItem1–4, checkboxLabel, confirmBtn, cancelBtn)

## Key Design Decisions

1. **No "pending consent" state**: Cancelling the consent modal simply closes it — the child is not created at all. This is the correct COPPA behavior: an account that was never created has no pending state to display. A pending-consent UI would only make sense if the child were created first and consent collected separately (deferred consent), which is not this flow. Comment in code documents this for future contributors.

2. **FamilyManager does its own insert**: The component does not call `addChildToFamily` from onboarding-api.ts; it has its own direct `supabase.from('family_members').insert()`. `addChildToFamily` was also updated to pass `consent_given` so both insertion paths are consistent (onboarding wizard + settings panel).

3. **Inline styles for modal overlay**: Consistent with `InstallPrompt` and `OfflineBanner` — ensures the overlay renders correctly before Tailwind CSS hydration.
