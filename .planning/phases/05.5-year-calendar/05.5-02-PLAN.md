## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| child (browser) → /api/wallet/award | Untrusted `{childId, date}`; the child controls the date string and could target another child. |
| child (browser) → streaks table | Historically writable via DevTools/anon client — the core CR-01 hole. |
| server action (repair-achievements) → streaks | Runs privileged; must use service-role, not the sessionless anon singleton. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05.5-04 | Elevation of Privilege | streak bonus in /api/wallet/award | mitigate | Streak-bonus intent gated on `date === localDateString()` (server clock) — a child cannot loop distinct past-date strings to mint bonus×N (D-12.1). Other backfillable intents remain unaffected. |
| T-05.5-05 | Tampering | streaks table writes | mitigate | All streak writes move to the service-role admin client inside the guarded award route / server action; combined with the SELECT-only RLS lock applied in 05.5-07, the client can no longer write current_count (D-12.2/D-12.3). |
