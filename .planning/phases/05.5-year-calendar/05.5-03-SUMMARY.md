## Accomplishments
- Four regional vacation preset JSON files (RU quarters, RU Moscow trimesters "5+1", KZ, BY), each covering both the 2026-27 and 2027-28 school years, plus a typed `index.ts` barrel exporting `PRESETS`

- `VacationPeriod.preset_id` added as a first-class, nullable, editable field — manual entry via `PeriodsManager` remains untouched and equally first-class

## Decisions Made
- Declared `VacationPreset`/`PresetPeriod` types in `data/vacation-presets/index.ts` rather than `lib/vacation-presets.ts` as the interfaces block in the plan suggested literally — the plan's own module-dependency arrow (`lib/vacation-presets.ts` → `data/vacation-presets/*.json`) combined with `applyPreset` needing `PRESETS` from the index would otherwise create `lib/vacation-presets.ts` ⇄ `data/vacation-presets/index.ts` circularity. `lib/vacation-presets.ts` re-exports both types via `export type { VacationPreset, PresetPeriod }`, so the public interface named in the plan (`import { applyPreset, listPresets } from '@/lib/vacation-presets'` plus the types) is unchanged for any consumer.
- Cast `PRESETS` to `VacationPreset[]` in `index.ts` — TypeScript widens JSON-imported string literals (`"quarters"`) to `string`, which doesn't satisfy the `TermMode` union on its own. The cast is safe: Task 1's automated verify gate independently confirms `ru-moscow-trimesters.json` has `termMode: "trimesters"` and the other three have `"quarters"` at the JSON source.
- Used typical/plausible (not authoritative) vacation dates per the plan's explicit "these are a template" guidance (D-01 specifics) — actual dates vary by school and are expected to be edited; this is documented in the phase CONTEXT.md ("типовой график").
