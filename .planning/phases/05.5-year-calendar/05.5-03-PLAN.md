<objective>
Ship the bundled regional vacation preset directory and the service that materializes a preset into the existing `vacation_periods` table. Presets cover Russia (traditional quarters + Moscow trimesters/modules), Kazakhstan, and Belarus for the current and next school year. Applying a preset inserts marked rows (`preset_id`); re-applying offers replace-vs-add-missing. Manual entry via the existing PeriodsManager stays a first-class equal path.

Purpose: Delivers SC2 (region fills vacations; manual value wins). "Manual wins" falls out naturally because materialized rows are edited/deleted like any other row, and replace only touches preset-marked rows.

Output: `data/vacation-presets/*.json` + loader index, `lib/vacation-presets.ts`, and a `preset_id` field on the `vacation-api.ts` type/CRUD.
</objective>
