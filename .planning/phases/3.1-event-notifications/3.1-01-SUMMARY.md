## Accomplishments
- Created reusable `notifyChild(childId, title, body, url)` server action that resolves child_id → member_id and fans out to all push subscriptions
- Hooked `notifyChild` into `approvePurchase` in wallet.repo.ts via dynamic import to avoid circular dependency
- Push errors silently swallowed at both layers — approval flow is never broken by notification failure
