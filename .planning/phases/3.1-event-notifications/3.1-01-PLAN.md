Create `app/actions/push-notifications.ts` as a `'use server'` file with a single exported function:

```typescript
export async function notifyChild(
  childId: string,        // children.id (NOT member_id)
  title: string,
  body: string,
  url: string
): Promise<void>
```

Implementation steps:
1. `createClient()` from `@/lib/supabase/server`
2. Query `family_members` to resolve `child_id` → `member_id`:
   ```typescript
   const { data: member } = await supabase
     .from('family_members')
     .select('id')
     .eq('child_id', childId)
     .maybeSingle()
   if (!member) return  // child not linked to a member yet — silent fail
   ```
3. Query `push_subscriptions WHERE member_id = member.id`
4. If no subscriptions, return silently
5. `await Promise.allSettled(subs.map(row => sendPushToSubscription(JSON.stringify(row.subscription), title, body, url)))`

Import `sendPushToSubscription` from `@/app/actions/push`.

Error policy: wrap in try/catch, log errors with `console.warn('[notifyChild]', err)` but never throw — push failures must never break the calling flow.

```typescript
// Fire push notification to child — non-blocking, errors swallowed inside notifyChild
try {
  const { notifyChild } = await import('@/app/actions/push-notifications')
  await notifyChild(
    purchase.child_id,
    `Покупка одобрена! ${purchase.reward_icon ?? '🎁'}`,
    `${purchase.reward_title} — заслужил!`,
    '/kid/wallet'
  )
} catch (e) {
  console.warn('[approvePurchase] push failed:', e)
}
```

Use dynamic import (`await import(...)`) to avoid circular dependency issues since `wallet.repo.ts` is not a server action itself. The dynamic import is safe because `approvePurchase` is only called from server contexts (parent dashboard page).
