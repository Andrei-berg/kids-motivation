# Phase 4.2: ux-polish - Context

**Gathered:** 2026-05-15
**Status:** Ready for planning

<domain>
## Phase Boundary

Make every page feel instant and every tap target easy to hit on a phone. Scope:
1. Framer Motion page transitions (eliminate hard flash on navigation)
2. Skeleton loaders on data-heavy screens (eliminate layout shift)
3. 44×44px touch targets throughout the app (eliminate missed taps)
4. Animation personality — expressive delight on both parent and kid screens

Scope does NOT include: new features, push notification changes, data model changes, new screens.

</domain>

<decisions>
## Implementation Decisions

### Page Transitions
- Style: Fade + slight scale — pages fade in while scaling from 98→100%
- Duration: 150–200ms (very quick — matches existing `--transition-fast` CSS variable)
- Scope: All routes — wrap root layout so every navigation gets the transition
- Reduced motion: YES — respect `prefers-reduced-motion`; Framer Motion handles this natively (transitions become instant)

### Skeleton Loaders
- Accuracy: Pixel-accurate — skeletons exactly mirror real component structure (no layout shift when data arrives)
- Priority screens (ALL required): Dashboard, Kid day screen, Wallet/transactions, Parent center pages
- Animation: Both modes
  - Shimmer (existing `@keyframes shimmer` in globals.css) for content areas (cards, sections)
  - Pulse (opacity fade) for small inline elements (badges, coin counts)
- Theme-aware: Dark grey skeletons on parent dark background; warm grey on kid warm-cream background

### Touch Targets
- Minimum: 44×44px everywhere — no exceptions
- Method: Invisible padding / `::after` pseudo-element — visual button size unchanged, tap area extended
- Nav pills exception: Give pills a VISIBLE size bump to ~44px height (intentional visual improvement, not just tap area)
- Tap feedback: Add subtle ripple effect on tap (especially important for grade buttons kids tap repeatedly)

### Animation Feel
- Parent dashboard (dark theme): Expressive — visible delight moments; coin balance counts up/down, data loads with bounce, number counters animate
- Kid screens (warm theme): Even more expressive than parent — achievement cards pop in, coins feel rewarding
- Coin balance changes: Count-up animation on wallet balance update (both parent and kid views)
- List items: Stagger in — items fade+slide in with 50ms stagger per item (transactions, achievement cards, leaderboard entries)

### Claude's Discretion
- Exact ripple implementation (CSS only vs. small JS handler)
- Stagger easing curve and direction (fade+translateY vs. fade+translateX)
- Count-up duration and easing for coin balance
- Whether to use Framer Motion's AnimatePresence or a custom CSS approach for the root layout wrapper

</decisions>
