## Decisions Made
- Kept ROOM_ITEM_LABELS and MOOD_OPTIONS as module-level constants with `labelKey` property; labels resolved via `t(labelKey)` inside component. This avoids hook calls at module level while keeping the static structure clean.
- `todayLabel()` refactored to accept `language` parameter and use `new Date().toLocaleDateString(locale, {...})` — eliminates hardcoded Russian day/month arrays entirely.
- DB identifier strings (`'цифровое задание'`) not translated — they function as database keys, not UI labels.
