**Why human:** ZIP blob download requires browser interaction and a live authenticated Supabase session.

#### 2. Danger Zone Data Summary and Download-First Link

**Test:** In Settings → Account tab → Danger Zone, observe the data summary block and the "download your data first" link.
**Expected:** Data summary shows real counts (not zeros or errors). The "download your data first →" link is visible above the type-to-confirm input and triggers a download when clicked.
**Why human:** Requires `/api/family-summary` to return real counts from a live authenticated session.

#### 3. Type-to-Confirm Delete Gate

**Test:** Type partial text (e.g. "DEL") then the full word "DELETE" into the confirm input in the Danger Zone.
**Expected:** Delete button remains disabled for partial input. Becomes enabled only on exact match "DELETE".
**Why human:** Interactive form state — requires browser.

#### 4. COPPA Consent Gate Flow

**Test:** In Settings → Family Manager, add a child with birth year set to `current year - 8` (age 8).
**Expected:** A full-screen consent modal appears (not the normal creation flow). Modal lists 4 data categories collected. Clicking Cancel closes the modal and no child is created. Checking the checkbox and clicking "Confirm & Create Account" creates the child.
**Why human:** Requires browser interaction and live Supabase insert to verify child creation/non-creation.

#### 5. Audit Log Screen and Event Recording

**Test:** Approve or reject a pending shop purchase in ParentCenter → Shop tab. Then navigate to ParentCenter → Audit tab.
**Expected:** The Audit screen loads, shows the approve/reject event with a human-readable description (e.g. "Approved shop purchase: Minecraft (50 coins) for Adam"), child filter pills work, and coin delta is shown in green/red.
**Why human:** Requires live `parent_audit_events` table (SQL migration must be applied to Supabase) and real pending shop purchases.

**Note for human tester:** The SQL migration `supabase/migrations/04.4-01-audit-consent.sql` must be run manually in the Supabase SQL Editor before any audit event recording or consent_given storage will work. Without this migration, the audit log and COPPA gate will silently fail at the database layer.
