  <action>
Create components/parent-center/screens/AuditScreen.tsx:

Structure:
- Props: `{ familyId: string; children: ParentChild[] }` (children passed from ParentCenter for filter labels)
- State: events (AuditEvent[]), loading (bool), filterChild ('all' | string), loadingMore (bool), offset (number)
- Load: useEffect → getAuditEvents(familyId, { childId: filterChild !== 'all' ? filterChild : undefined, limit: 30 })
- Re-fetch when filterChild changes (reset offset to 0)
- "Load more" button: appends next 30 with offset increment

UI layout (all inline styles using T tokens, matching ParentCenter dark theme):
1. Header: title "Audit Log" using T.fHead + subtitle using T.muted
2. Child filter pills: "All" + one pill per child in props.children — active pill uses T.indigo background
3. Entry list: each AuditEvent renders as a row:
   - Left: action icon based on action_type (🛍️ shop_approve, ❌ shop_reject, 💰 coin_adjust, 🏆 badge_award, ⚙️ settings_change, 📤 data_export, 🗑️ account_delete_request)
   - Middle: description text (T.text, 14px) + timestamp below (T.muted, 12px, formatted as "May 17, 14:32")
   - Right: coins_delta shown if non-null (green for positive, red for negative, T.fMono)
4. Empty state: "No audit events yet" (T.muted, centered)
5. Loading: 3 shimmer skeleton rows (div with T.cardHi background, animated opacity pulse via CSS)
6. "Load more" button at bottom if events.length === current limit (30)

All text strings via useT():
- t('audit.title'), t('audit.filterAll'), t('audit.loadMore'), t('audit.noEvents'), t('audit.loading')
- Timestamp formatting: use date-fns format(new Date(event.created_at), 'MMM d, HH:mm') — date-fns is already in package.json

Do NOT use Framer Motion in this component — the audit list is functional, not animated.
