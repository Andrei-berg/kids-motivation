# Phase 4.5 Plan 02: Parent Center Screens Desktop Layouts Summary

Dashboard, Analytics, ChildrenTasksShop (Children + Shop), and Settings now render purposeful 2-column (or nav+content) desktop grids at >=1024px instead of single-column mobile cards stretched inside the wider Plan-01 shell — closing DSK-02.

## What Was Built

### Task 1 — Dashboard: 2-column grid, children left / activity right (commit: 1a6420d)

- Added `useDesktop()` hook (window resize listener, 1024px breakpoint)
- Header, "today filled" gradient card, and reading-checks section stay full-width above the grid (unchanged JSX, just sit above the new grid wrapper)
- New `gridTemplateColumns: isDesktop ? '1fr 1fr' : '1fr'` wrapper below that:
  - Left column: "Children" section header + `ChildCard` list (unchanged sub-component)
  - Right column: pending-approvals section (conditional) stacked above the activity feed card — grouped together as the "attention/updates" column
- Padding bumps to `24px` uniform on desktop vs the mobile `20px 16px 24px`
- No changes to `ChildCard`, `ActivityRow`, or any prop/data flow — pure JSX restructuring

### Task 2 — Analytics, ChildrenTasksShop, Settings: multi-column desktop grids (commit: 9016b58)

**Analytics.tsx**
- Added `useDesktop()` hook
- KPI mini-card grid: `repeat(2,1fr)` mobile → `repeat(4,1fr)` desktop
- Bar chart card ("Coins earned vs spent") and line chart card ("Task completion · children") now sit in a `1fr 1fr` grid on desktop (was stacked full-width cards)
- "Children overview" list card uses `gridColumn: isDesktop ? 'span 2' : undefined` — stays full-width since it's a row list, not a paired visualization
- Padding bumps to `24px` on desktop

**ChildrenTasksShop.tsx**
- Added one shared `useDesktop()` hook used by both exported screens (`ChildrenScreen`, `ShopScreen`); `TasksScreen` untouched (mock-data screen, not in scope)
- `ChildrenScreen`: child profile cards (avatar, level, stats grid, goal bar) now render in a `1fr 1fr` grid on desktop instead of one stacked column
- `ShopScreen`: rewards/stickers list (`tab === 'items' || 'stickers'`) and the pending-purchase queue (`tab === 'queue'`) both get `1fr 1fr` grids on desktop; the three empty-state cards (loading spinner, "no rewards", "no pending requests") get `gridColumn: 'span 2'` so they render full-width rather than half-width orphans

**Settings.tsx**
- Added `useDesktop()` hook
- Desktop: root tab area becomes a `200px 1fr` grid — left column is a vertical button list built from the existing `tabs` array (family/coins/children/schedule/account), right column renders `content[tab]` unchanged
- Mobile: identical original JSX (sticky horizontal `Tabs` pill bar + `content[tab]` below), now inside the `!isDesktop` branch of a ternary — no behavioral or visual change on mobile
- Individual tab bodies (`FamilyTab`, `CoinsRulesTab`, `ChildrenTab`, `ScheduleTab`, `AccountTab`) were **not** touched — their internal `flexDirection:'column'` card stacks are unchanged; the desktop win here is the nav no longer eating a full pill-bar row and the content column being naturally narrower (viewport − sidebar − chat − 200px nav) instead of full-bleed

## Deviations from Plan

None — plan executed as written, taking the plan's own "simpler acceptable approach" language for Settings at face value (left-nav + content grid, not touching every tab's internal card stacking).

**1. [Rule 2-adjacent, low-risk addition] Analytics KPI grid widened to 4 columns on desktop**
- **Found during:** Task 2
- **Reasoning:** Plan's must-haves only mention chart cards, but leaving the KPI mini-cards at `repeat(2,1fr)` on a ~900px-wide desktop column would have produced two oversized, mostly-empty cards — directly contradicting the plan's stated purpose ("purposefully multi-column on desktop"). Bumping to `repeat(4,1fr)` is additive-only (`isDesktop ? 4 : 2`), does not touch mobile, and required no new component.
- **Files modified:** `components/parent-center/screens/Analytics.tsx`
- **Commit:** 9016b58
