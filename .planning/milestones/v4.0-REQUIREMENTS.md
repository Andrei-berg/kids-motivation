## v4.0 Requirements

Requirements for v4.0 PWA Polish milestone. Each maps to roadmap phases.

### PWA

- [x] **PWA-01**: User can install the app on iPhone/Android via Add to Home Screen prompt — Phase 4.1
- [x] **PWA-02**: Web Push notifications are delivered when the app is closed (service worker background push) — Phase 4.1
- [x] **PWA-03**: Basic offline support — cached shell loads without internet; graceful degradation shown — Phase 4.1

### UX Polish

- [x] **UX-01**: Pages transition with Framer Motion animations; skeleton loaders shown during data fetch (no layout shifts) — Phase 4.2
- [x] **UX-02**: All interactive elements are ≥44px touch targets throughout the app — Phase 4.2

### Localization

- [x] **LOC-01**: App is available in Russian and English; language auto-detected from browser with manual switcher — Phase 4.3

### Security / Compliance

- [x] **SEC-01**: User can delete account with full data cascade; data export available; consent gate shown for children under 13 (COPPA/GDPR) — Phase 4.4
- [x] **SEC-02**: Parent actions are recorded in an audit log (shop confirmations, settings changes, coin adjustments) — Phase 4.4

### Desktop

- [x] **DSK-01**: App has a dedicated desktop layout (≥1024px): sidebar navigation, multi-column content areas, full-width chat — Phase 4.5
- [x] **DSK-02**: Parent Center pages are fully designed for wide screen — no stretched mobile cards — Phase 4.5
- [x] **DSK-03**: Kid Screen pages are fully designed for wide screen — no stretched mobile cards — Phase 4.5

**Coverage:** 11/11 requirements complete.

**Note:** UX-02 and DSK-01 were found still marked "Pending" in REQUIREMENTS.md's traceability table at v5.0/v4.0 close time (2026-07-23) despite their phases (4.2, 4.5) being code-complete since 2026-05-16/18 — a doc-sync gap, not a functional gap. Corrected during this archival after confirming both in code (44px min-height rules present in globals.css; sidebar nav + useDesktop ≥1024px breakpoint present in components/parent-center/).

### Out of Scope (v4.0)

| Feature | Reason |
|---------|--------|
| Apple ID login | Google + email sufficient for beta |
| Video messages | High storage/bandwidth cost |
| Offline Realtime chat | Real-time is core; graceful degradation is sufficient |
| B2B teacher/coach accounts | Post-product-market-fit |
| Freemium limits + Stripe | Deferred to v6.0 |
