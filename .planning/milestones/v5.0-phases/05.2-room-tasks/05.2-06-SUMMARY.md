## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing Critical] Added `common.moveUp`/`moveDown`/`enable`/`disable` i18n keys**
- **Found during:** Task 1 (component build)
- **Issue:** The plan only specified `settings.tabs.room` as a required i18n key. While building the component I initially used `t(key) || 'English fallback'` for button tooltips and empty-state text, but `useT()`'s missing-key behavior returns the key string itself (not `undefined`/`''`), so `||` fallbacks never fire — Russian users would see raw keys like `common.moveUp` instead of any text.
- **Fix:** Added `settings.roomTasksManager.*` (title, subtitle, namePlaceholder, empty, systemTask, deleteConfirm) and `common.moveUp/moveDown/enable/disable` as real keys to both `en.json` and `ru.json`, then removed the dead `|| 'fallback'` expressions from the component.
- **Files modified:** components/settings/RoomTasksManager.tsx, public/locales/en.json, public/locales/ru.json
- **Verification:** `npx tsc --noEmit` clean; manual read-through of every `t()` call confirms a matching key exists in both locale files.
- **Committed in:** `5c6cf06` (Task 1 commit) and `c73609d` (Task 2 commit, for the tabs.room key)
