# Phase 05.2 Plan 04: KidDayFillForm renders room_tasks + dual-write Summary

**Kid day-form Room checklist is now data-driven from `room_tasks` (active, ordered) instead of a hardcoded 5-item array, and `handleSubmit` dual-writes `room_checks` (every rendered task) plus the legacy `days.room_*` columns for tasks carrying a `legacy_key`, keeping the room trigger/room_ok/analytics/wallboard unchanged.**

## Accomplishments
- Removed the hardcoded `RoomItems` interface and `ROOM_ITEM_LABELS` array; the Room `FillSection` now maps over `roomTasks` (`getRoomTasks(familyId)`, active, `sort_order` ascending), rendering `task.icon`/`task.name` directly (no i18n labelKey lookup for these — matches CONTEXT decision 6).
- Added `roomTasks`/`roomChecked` state (`roomChecked` keyed by `task.id`). Loaded in the existing mount `useEffect`: `family_id` is read from the same `children` query already fetching `require_reading_check` (no extra round trip); `roomChecked` is pre-filled from `getRoomChecks(childId, date)` only when `isChildFilled` (mirrors the prior `existingDay?.room_*` pre-fill gate) — otherwise all loaded tasks default to unchecked, matching prior parent-filled-day behavior.
- `coinsPreview`'s room contribution now uses `roomDoneCount >= max(1, ceil(0.6*roomTaskCount))` — byte-parity with the Plan 03 server threshold for the default 5-task family (3/5) — still a client preview only; `/api/wallet/award` remains the sole source of credited coins.
- `handleSubmit` dual-write: builds `legacyDone` (all 5 `RoomLegacyKey`s explicitly defaulted `false`, then set from `roomChecked` for any task with a matching `legacy_key`) and passes it into `saveDay`'s `roomBed/roomFloor/roomDesk/roomCloset/roomTrash` params (unchanged call site), then calls `saveRoomChecks(childId, date, roomTasks.map(...))` immediately after, writing one row per rendered task. A legacy task that's deactivated/not rendered now explicitly writes `false` rather than leaving the param `undefined` (which would let `saveDay`'s `params.roomX ?? roomData?.x ?? false` merge resurrect a stale prior value).
- All other sections (mood, activities, grades, exercises, sections, reading, proof upload, `/api/wallet/award` POST, streaks, badges) left untouched, per the plan's scope.
