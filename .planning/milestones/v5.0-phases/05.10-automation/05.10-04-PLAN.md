<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| parent browser → server actions (setTrustLimitAction/setAllowanceAction) | untrusted numeric payload crosses here; must be parent-guarded + clamped |
| client → children.* columns | ordinary (non-money) columns; still writable only via parent-guarded service-role action |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-0510-13 | Tampering | server-action numeric inputs | mitigate | `clampTrustLimit`/`clampAllowance` server-side (floor, NaN-drop, range clamp, period whitelist) before every `.update()` — independent of any UI constraint |
| T-0510-14 | Elevation | writing another family's child | mitigate | `requireParent()` + `assertChildInFamily(admin, childId, member.familyId)` gate every action |
| T-0510-15 | Spoofing | client writing children.* directly | mitigate | Writes go through service-role server actions under a parent guard; the browser client cannot set trust_limit/allowance to bypass review |
| T-0510-16 | Information Disclosure | cards reading children row | accept | Cards read only the target child's own scalar settings via the existing family-scoped read path; no cross-child exposure |
</threat_model>
