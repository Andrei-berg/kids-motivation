**CoinsRulesTab now has 3 rule-preset cards with a mandatory old->new diff-preview before any write, a 3-way grade-scale Tabs driving data-driven grade_coin_map rows, and the Behavior Tags manager + Behavior approval queue mounted underneath ‚Äî all writes still go exclusively through `updateWalletSettingsApi` (server PATCH). ## Accomplishments
- **Task 1 ‚Äî Preset cards + diff-preview + apply:** `PRESET_IDS` render as 3 equal-weight cards (Classic / No Penalties / Bonuses Only) using the existing `ChildrenTab` card-select idiom (indigo border + glow when selected). "Apply preset" (`Btn variant="primary"`) stays `disabled` until a card is picked. Clicking it opens a diff-preview panel ‚Äî `Object.entries(getPresetValues(id))` filtered to only the fields that actually differ from the loaded `settings`, each rendered as an icon + label + old `Amount` (muted, non-money) ‚Üí new `Amount` (gold if non-negative, danger-colored if negative, `signed`). "Apply these changes" calls `updateWalletSettingsApi({ ...settings, ...patch })` and reloads local state from the server response; "Keep current rules" dismisses with zero writes. No direct client `supabase.from('wallet_settings')` write exists anywhere in the file.
- **Task 2 ‚Äî 3-way grade-scale Tabs + data-driven rows:** The fixed 5-row grade block (`RuleRow` x5 for `coins_per_grade_1..5`) is gone. A `Tabs` (reusing Settings.tsx's already-imported ink-native `Tabs` from `../ui` ‚Äî no second `Tabs` implementation introduced) switches `settings.grade_scale` between `five_point`/`twelve_point`/`a_f`. Switching seeds `grade_coin_map` from `defaultGradeCoinMap(scale)` only when the new scale's keys are missing (D-08: forward-looking only ‚Äî a family's already-recorded grades keep their original literal coin value; nothing is ever locked or rewritten), and shows `settings.gradeScale.switchNotice` as a toast (`notify()`), not a blocking modal. Below the tabs, `GRADE_SCALE_VALUES[grade_scale].map(...)` renders one row per value, each a click-to-edit `Amount` (display mode) that swaps to a number input on click, mirroring the exact interaction shape `BehaviorTagsManager` (plan 05) already established for its price field. Saving (the existing bottom "Save rules" button) persists `grade_scale` + `grade_coin_map` through the same `updateWalletSettingsApi` call the rest of the tab already used ‚Äî no new save path.
- **Task 3 ‚Äî Mount BehaviorTagsManager + BehaviorApprovalQueue:** Both plan 05/07 components are imported and rendered bare-mounted (each inside its own `Card pad={16}`, matching the visual language of the rest of `CoinsRulesTab`) in a wrapping `div` with `marginTop: 24` (the UI-SPEC's `lg` token) directly below the "Save rules" button, so they read as a distinct "Behavior" section without duplicating either component's internal CRUD/approval logic.
- Added the small set of i18n keys (en + ru) these three tasks structurally needed but plan 02's earlier pass didn't cover: `settings.rulePresets.title`/`diffNoChanges`, `settings.gradeScale.rowLabel`, and `settings.coinRules.coachGrade2`/`coachGrade1`/`book` (needed by the diff-preview's field-label map since D-02's preset scope includes those three fields but no prior UI row/label existed for them).

## Decisions Made
See `key-decisions` in frontmatter above. The two most significant:
1. The diff-preview's field-label map needed three i18n keys (`coachGrade2`, `coachGrade1`, `book`) that never existed before, because the pre-existing `CoinsRulesTab` never rendered a row for `coins_per_coach_2`, `coins_per_coach_1`, or `coins_per_book` even though `lib/presets.ts`'s D-02 field scope (plan 02) always included them. Without these labels the diff-preview would either crash or fall back to a raw field-name string for exactly the fields a family is least likely to have manually tuned before.
2. Grade-coin-map rows use the same click-to-edit `Amount` interaction `BehaviorTagsManager` established for its price field (plan 05), rather than the old always-visible-input `RuleRow` pattern, so every coin value in this phase's new UI satisfies "Coin values render via Amount (no raw span coin values)" while staying editable.
