## Sampling Rate

- **After every task commit:** Run `npm test`
- **After every plan wave:** Run `npm run test:integration`
- **Before `/gsd:verify-work`:** Full suite must be green, plus `npx tsc --noEmit` clean (critical — the `grade: number → string` type change touches ~6 files)
- **Max feedback latency:** 180 seconds

> **Accepted latency exception (Dimension 8):** the behavior/award integration tests (Plan 06 Task 3, Plan 07 Task 3) are live-DB (~2-3 min), knowingly exceeding both the 30s Nyquist WARNING threshold and this phase's own 180s target. Live-DB round trips are unavoidable for award economics, idempotency, the D-09 legacy fallback, and the approve→credit boundary. The sub-30s sampling lane is `npm test` (unit — `tests/presets.test.ts`); these integration suites run per-wave, not per-task-commit. This is a documented, accepted tradeoff, not a violation to fix.
