# Phase 05.9 Plan 04: grade_coin_map read/write plumbing Summary

**grade_coin_map is now always readable (server + client, fallback-derived from coins_per_grade_1..5) and always safe to write (shape/bounds-validated on the settings PATCH before upsert).**

## Accomplishments
- `loadSettings` (server, `app/api/wallet/_lib.ts`) always returns a populated `grade_coin_map`, deriving it from the family's own `coins_per_grade_1..5` when the JSONB column is null — no migration backfill.
- `getWalletSettings` (client, `lib/repositories/wallet.repo.ts`) gains the identical fallback via a new `withGradeCoinMapFallback` helper, applied uniformly to both the DB-row branch and the hardcoded-defaults branch.
- `clampGradeCoinMap` added to `PATCH /api/wallet/settings` — rejects non-object shapes, rejects the entire map on any non-finite value, clamps finite values to `[-100000, 100000]` — mirroring `clampStreakFields`' defense-in-depth idiom.
