**Net-new read-only `CalendarGrid` component — Mon-first month grid with day-type coloring (school/weekend/vacation/sick), month nav, child filter, and a per-child sick-day dot overlay with tap popover — composed as the hero of the `calendar` sub-tab above the two existing (already-recolored) editors, now collapsed accordions.**

## Accomplishments
- `CalendarGrid.tsx` renders a Mon-first month grid built with `date-fns` (`startOfMonth`/`endOfMonth`/`eachDayOfInterval`/`getDay`), colored per day type via `getDayType()` + `getFamilyCalendar()` + `getVacationPeriods()`, with a 2px indigo inset ring on today's cell
- Month nav (prev/next via `Icon chevL/chevR` with aria-labels, "Today" reset `Btn`) and a child-filter `Tabs` row (Family vs per-child) drive both the day-type fill and which `getDaysInRange` reads are fetched
- D-06 sick overlay: up to 3 colored dots (from `CHILD_ACCENTS`, 5px/2px-gap/bottom-right/4px-inset) per cell in family-wide view, collapsing to 2 dots + `+N` label beyond 3; tapping a dotted cell opens a cell-anchored popover listing sick child name(s) via the `sickOne`/`sickMany` i18n keys; single-child view instead shows the full-cell danger tint (no dots)
- `Settings.tsx`'s `calendar` sub-tab restructured: `CalendarGrid` as the hero `Card`, with `CalendarSettingsManager` and `PeriodsManager` demoted to two collapsed `AccordionSection` disclosures below it (local component, T-token styled, default collapsed) — no new top-level nav item or sub-tab id added; `vacations` sub-tab kept standalone as before
- Read-only throughout: no insert/update/delete call added anywhere in `CalendarGrid.tsx`; `is_sick` is still only ever set from the day-fill forms
