### `getDaysInRange(childId, start, end)` (new repo function)

```typescript
// Source pattern: lib/repositories/children.repo.ts:496-506 (days query only)
export async function getDaysInRange(childId: string, startDate: string, endDate: string) {
  const { data, error } = await supabase
    .from('days')
    .select('*')
    .eq('child_id', childId)
    .gte('date', startDate)
    .lte('date', endDate)
    .order('date')
  if (error) throw error
  return data ?? []
}
```
Place in `lib/repositories/calendar.repo.ts` (co-located with
`getFamilyCalendar`) ŌĆö that file's header already documents "client-write
path, no admin client" which applies identically here. Do NOT import
