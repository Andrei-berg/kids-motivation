    - Custom settings: streak_study_days=2, streak_study_bonus=250, study count 2 → 250.
    - Clamp (lower): streak_*_days stored as 0 or negative is treated as 1 (threshold ≥ 1); streak_*_bonus stored negative is treated as 0 (never a negative award).
    - Clamp (upper): streak_*_bonus stored above 100000 (e.g. 9999999) is capped at 100000 (no unbounded/absurd award).
  </behavior>

    parameter (`settings: WalletSettingsRow` / `Record<string, number>`) and DELETE its

    room → if `current_count >= streak_room_days` add `streak_room_bonus`; study →
    `streak_study_days` / `streak_study_bonus`; sport → `streak_sport_days` /
    `streak_sport_bonus`. Apply clamps inside the function so stored garbage can't corrupt
