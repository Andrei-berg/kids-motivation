# Phase 05.7 Plan 04: Day-form block renderers + confetti removal Summary

**Decomposed the 1417-line KidDayFillForm into 4 family-bank block renderers (Tick micro-ticks, LedgerRow prices) behind a shared BlockCard receipt shell, removed all 6 per-toggle confetti calls plus the Confetti mount, and put ScreenHeader with logout on the day screen — with the /api/wallet/award save flow byte-identical.**

## What was built

### Task 1 — Block renderers (commit 1c36a9f)
- `BlockCard.tsx`: receipt-style card (paper.card, hairline rule under an 18px/700 Bitter title, 16px padding) — the visual replacement for the old `FillSection`.
- `RoomBlock.tsx`: 2-col room-task grid; each row toggles with the `Tick` micro-tick (≤200ms draw-in, reduced-motion safe). Photo-proof capture UI carried over, restyled to paper tones.
- `ActivitiesBlock.tsx`: each activity is a `Tick` + `LedgerRow tone="earn"` price row (gold mono amount only when the activity pays coins).
- `BookBlock.tsx`: reading toggle, book/pages/minutes/bookmark inputs, "finished" toggle with `Tick`; pending-verification note in `warningText`.
- `CustomBlock.tsx`: generic toggle with `Tick` + `LedgerRow` (`earn`/`penalty` by price sign); `who_fills === 'parent'` stays read-only (T-056-12 UI guard preserved).
- Zero `triggerConfetti` in the directory; no coin value hardcoded — all amounts come from props/config rows.

### Task 2 — Wiring + day page (commit d0aa505)
- `KidDayFillForm.tsx`:
  - `roomBody`/`extraActivitiesBody`/`readingBody` now render the extracted components; `renderCustomBlock` wraps `CustomBlock` in `BlockCard`. The `assembleDayBlocks`-driven selection switch and all toggle state/callback identities are unchanged (D-06/5.6 seam intact).
  - All 6 `triggerConfetti()` toggle calls, the `<Confetti trigger={confettiTrig}/>` mount, the `confettiTrig` state, and the `Confetti` import removed (D-18).
  - Flag-off legacy path deduped onto the shared bodies (grades/exercises/sections/reading/activities) — file went from 1417 to ~1030 lines.
  - Hero reskinned to a paper receipt card ("≈ +N" gold mono estimate); mood row and save CTA (indigo `paper.accent`) on tokens.
  - `handleSubmit`'s `fetch('/api/wallet/award', { body: { childId, date } })` and `onSaved(coinsPreview)` verified byte-identical via git diff (Plan 11 owns the stamp wiring).
- `app/kid/day/page.tsx`: local `useDesktop` copy replaced with the shared `@/lib/hooks/useDesktop`; `ScreenHeader title={t('kidHeader.day')} showLogout` mounted on both mobile (replacing the old inline avatar header; XP bar + streak flame kept below) and the desktop right column; the 4.5 sticky-sidebar layout untouched (D-15). `onSaved` coins/confetti gating left as-is for Plan 11.
