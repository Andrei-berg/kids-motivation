## Decisions Made
- Tabs reads the theme's opaque card surface (`paper.card`/`ink.card`) directly from `lib/design/tokens` for the active-tab background rather than extending `resolve()` with a `card` field — no other atom in the file needs an opaque surface color, so this keeps `resolve()`'s return shape focused on text/tone colors.
- Tick's "on" state fill uses `success` (green), not `accent` (indigo) — a checked box is a status/completion signal, and UI-SPEC's color contract reserves indigo strictly for interactive/navigation elements.
- Created new top-level `wallet` and `kidHeader` locale namespaces (didn't exist previously) instead of overloading the existing `kidWallet` namespace, since the plan's verification script requires the exact key paths `wallet.emptyHeading`/`wallet.emptyBody`.
