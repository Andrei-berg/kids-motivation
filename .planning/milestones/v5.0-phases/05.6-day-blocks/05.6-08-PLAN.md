<objective>
Close CR-02 at the database layer and prove it, plus fold in the two cheap
money/security warnings that live on the same surface (WR-01 RLS half, WR-07 seed
guard). Three things must be true after this plan:

1. **Defense-in-depth against UUID rotation:** child-role sessions lose DELETE on
   `day_block_entries` (the forms only ever upsert, never delete), so the rotation
   primitive behind CR-02 is removed even beyond the deterministic key from Plan 07.
2. **Idempotency continuity on the key-format switch:** existing `custom_block`
   `wallet_transactions` rows were keyed to the entry UUID (old code). Plan 07
   switches the route to `${date}:${block_id}`. Without a backfill, an already
   awarded day/block would look un-credited under the new key and could be
   re-awarded once. Backfill rewrites those rows to the new format.
3. **WR-01 / WR-07:** tie `day_block_entries` INSERT/UPDATE `child_id` to the
   family via WITH CHECK, and gate `seed_default_day_blocks` to family members.

Then extend `verify-day-blocks-award.mjs` to exercise the real exploit path
(entry rotation), not just a literal duplicate source_id.

Purpose: SC3 / D-09 idempotency must be provably un-rotatable and continuous.
Output: a new applied migration + a verify probe that fails if the exploit reopens.

Depends on Plan 07 because the backfill target format and the probe's award key
must exactly match the route's chosen `${date}:${block_id}` source id.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| child auth token → PostgREST (day_block_entries) | Family-wide RLS lets a child insert/delete/re-insert entry rows. DELETE is the rotation primitive behind CR-02; cross-family child_id injection is WR-01. |
| any authenticated user → seed_default_day_blocks RPC | SECURITY DEFINER + GRANT authenticated lets any logged-in user seed any family's money-adjacent config (WR-07). |
| key-format switch (Plan 07) → wallet_transactions history | Old rows keyed to entry UUID; unmigrated rows could be re-awarded once under the new natural key. |
