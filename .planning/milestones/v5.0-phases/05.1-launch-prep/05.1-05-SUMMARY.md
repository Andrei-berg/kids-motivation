## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug in test seed data] `room_ok` cannot be set directly on insert — it's DB-trigger-derived**
- **Found during:** Task 2, first `npm run test:integration` run
- **Issue:** The plan's `<behavior>` spec says to seed "a `days` row (room_ok true, good_behavior true)". A first attempt inserting `{ room_ok: true, good_behavior: true }` produced a passing insert but `room_ok` read back as `false` — `room_score_trigger`/`update_room_score()` (BEFORE INSERT/UPDATE trigger) recomputes `room_score` from `room_bed`/`room_floor`/`room_desk`/`room_closet`/`room_trash` and sets `room_ok := room_score >= 3` on every write, overriding any client-supplied `room_ok` value.
- **Fix:** Seed 3 of the 5 checklist booleans (`room_bed`, `room_floor`, `room_desk`: all `true`) instead of `room_ok` directly, so the trigger derives `room_ok = true` as intended.
- **Files modified:** `tests/integration/award.test.ts`
- **Verification:** Re-ran `npm run test:integration -- tests/integration/award.test.ts` — all 5 tests green, `room` transaction present with `coins_change: 3`.
- **Committed in:** `631a9a5` (Task 2 commit — the fix is baked into the initial test-file commit; no separate commit needed since the file wasn't committed until the working version was confirmed green)
