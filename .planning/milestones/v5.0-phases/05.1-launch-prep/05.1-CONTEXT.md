<decisions>
## Implementation Decisions

### Key rotation (P0)
- Supabase service-role key AND database password must be rotated; old credentials must stop working.
- Rotation itself happens in the Supabase dashboard — plan these as manual tasks (`autonomous: false`)
  with exact click-path instructions; code/env updates around them are agent work.
- After rotation: update `.env.local` locally and Vercel project env (`SUPABASE_SERVICE_ROLE_KEY`,
  `SUPABASE_DB_URL`); redeploy; run existing verify scripts against prod to confirm the app works.
- `.env.local` is git-ignored — never print new secrets into the transcript, files, or logs.

### Monitoring
- Sentry for error tracking (@sentry/nextjs), free tier: client + server + edge configs,
  source maps off in free plan if config burden is high — minimal setup is acceptable for this phase.
- Product analytics: PostHog (free tier, EU cloud preferred given RU/EU audience) — page views +
  a first custom event (e.g. `day_saved`). Lightweight wrapper so the vendor can be swapped.
- Both must be no-op safe when env keys are absent (dev without keys must not crash).

### Money integration tests
- Vitest. Tests hit a real Supabase project using the service-role key from env
  (same pattern as existing `scripts/verify-*.mjs`), using a dedicated **test family** created
  and torn down by the suite — never touch real family data.
- Must cover: award idempotency per `(child_id, source_type, source_id)` (double-POST same day →
  no double credit), each award source type (grades/room/behavior/sport/activities/book/streak),
  purchase freeze/approve/reject flow, exchange, withdrawal double-spend guard.
- Tests are skipped (describe.skipIf) when `SUPABASE_SERVICE_ROLE_KEY`/test env is absent, so
  `npm test` stays green for contributors without keys.
- Existing `scripts/verify-*.mjs` remain; the vitest suite formalizes them.

### Naming
- Product name: **FamilyCoins** everywhere user-visible: `app/manifest.ts` (or manifest source),
  layout metadata (title/description), README title. package.json name may stay `kids-motivation-v2`
  (private, not user-visible) — Claude's discretion.

### Claude's Discretion
- Sentry/PostHog wrapper file locations and init patterns (follow Next.js 14 App Router idioms).
- Test file layout under `tests/` and test-family fixture helpers.
- Whether to add a `/api/health` check for Sentry verification (a health route already exists).
</decisions>
