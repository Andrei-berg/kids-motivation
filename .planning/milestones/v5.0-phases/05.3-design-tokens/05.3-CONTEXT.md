<decisions>
## Implementation Decisions

### Palette (locked by design contract)
Base: brand indigo `#6C5CE7` (single interactive color), coin gold `#E9A83C` (money ONLY),
success `#2E9E77`, warning `#E07A2E`, danger `#D95563`.
- **paper (kid):** bg `#FAF3E9`, card `#FFFFFF`, ink `#241E38`, ink2 `#4A4363`, ink3 `#837C99`,
  line `#EBE2D2`, lineSoft `#F3ECDF`, accent `#5B4BD4` (indigo text-safe on paper), gold-text `#9C6B10`.
- **ink (parent):** bg `#141021`, bg2/card `#1D1830`, cardHi `#232040`-ish (planner derives),
  text `#EFEBFA`, textDim `#C9C3DE`, muted `#8F89AB`, accent `#8B7BF5`, gold-text `#F0BE6A`,
  danger-on-dark `#FF8A96`.

### Re-export mapping (the recolor mechanism)
- `components/kid/design/tokens.ts` keeps exporting `T` with the SAME KEYS (coral, coralDeep,
  coralSoft, teal, tealDeep, tealSoft, sun, sunDeep, sunSoft, plum, plumSoft, pink, pinkSoft,
  mint, ink, ink2, ink3, line, lineSoft, bg, card, success, warn, fDisp, fBody, fNum) but values
  from the new system: coral→indigo family (interactive), sun→gold family (money), teal→success
  family, plum stays indigo, pink/mint→muted supporting tints derived from the palette
  (planner picks tasteful mappings; nothing neon).
- `components/parent-center/tokens.ts` same: cyan→indigo `#8B7BF5`, neon success `#00E676`→`#2E9E77`
  (on-dark variant), warning `#FFD93D`→`#E07A2E` (on-dark-legible variant OK), danger `#FF6B6B`→`#FF8A96`,
  fonts remapped. CHILD_ACCENTS re-derived from the new palette (keep 5 entries, distinguishable).
- This intentionally recolors ALL kid + parent-center screens at once. Legacy pages (globals.css)
  are untouched and stay Inter-styled until 5.11.

### Fonts
- next/font/google: Bitter (display, weights 600–800), Golos Text (body 400–700),
  JetBrains Mono (numbers 500–700) — cyrillic + latin subsets, `display: swap`, CSS variables
  (e.g. --font-display/--font-body/--font-mono) wired in app/layout.tsx.
- Token font strings reference the CSS variables with system fallbacks. Old font names
  (Nunito, DM Sans, Sora) must not remain in token values. globals.css/Inter untouched.

### Shared atoms — new `components/design/atoms.tsx` (or lib/design/ — planner picks one home)
- `LedgerRow` — name + dotted leader + mono amount (the brand signature); tone variants:
  earn (gold), spend (neutral), penalty (danger), pending (muted, "≈" prefix support).
- `Amount` — JetBrains Mono, tabular-nums, gold for money; size variants.
- `StatusChip` — success/warning/danger soft chips.
- `useCountUp`/stamp animation — reuse the existing AnimatedNum/useCountUp approach (rAF cubic
  ease-out already in kid atoms); the "stamp" entrance (translateY+scale, ~450ms) as a tiny
  CSS/inline util; must respect prefers-reduced-motion.
- Theme-aware: atoms take colors from the theme (paper/ink) — a minimal ThemeContext or explicit
  `theme` prop, planner decides (simplest wins; no CSS-in-JS libs).
- PILOT adoption only: kid wallet page (`app/kid/wallet/page.tsx` history list → LedgerRow/Amount)
  and parent dashboard (`components/parent-center/screens/Dashboard.tsx` activity list → LedgerRow).
  Other screens keep their current markup (they still recolor via tokens).

### Verification bar (ROADMAP 5.3 SC)
1. tokens.ts exports base+two themes; old files re-export with unchanged keys (tsc proves no
   consumer breaks).
2. Fonts load via next/font with cyrillic subsets.
3. Atoms exist and are used on kid wallet + parent dashboard.
4. Unmigrated screens render without breakage — `npm run build` + visual checkpoint.

- Coin SVG/monogram component now or in 5.7 (defer OK).
- Whether AnimatedNum moves into shared atoms or is re-exported.
</decisions>
